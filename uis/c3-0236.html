1 · Objective

agentc2/packages/agentc2/src/skills/marketplace.ts + the Playbook, PlaybookInstallation, PlaybookPurchase, PlaybookReview, PlatformMarkup tables are the existing marketplace primitives; agentc2/apps/agent/src/app/api/ hosts the API routes. This epic completes the agent-registration + job-submission + result-upload API surface that extends the marketplace into a runnable Apex toolbox catalog. Off-theme note: this is C4 territory but built on C3 infra.

2 · Acceptance Criteria

  • POST /api/marketplace/agents registers an agent (slug, model, skills, owner, pricing) with playbook-style metadata.
  • POST /api/marketplace/agents/:slug/jobs submits a job; returns { runId, eta }.
  • GET /api/marketplace/agents/:slug/jobs/:runId returns status + result url.
  • POST /api/marketplace/jobs/:runId/result agent-side endpoint to upload result (signed token).
  • Billing hook: each successful job creates a PlatformMarkup row applied against the customer's PlaybookPurchase.
  • Rate-limit per (customer, agent) per 1 min; quota enforced via existing budget package.
  • OpenAPI spec emitted; consumed by future Apex SDK.
  • End-to-end smoke: a registered agent receives a job, completes it, customer is billed correctly.

3 · Architecture & Approach

API surface

POST /api/marketplace/agents                       // register
GET  /api/marketplace/agents                       // list (filterable)
GET  /api/marketplace/agents/:slug                 // detail
POST /api/marketplace/agents/:slug/jobs            // submit job
GET  /api/marketplace/agents/:slug/jobs/:runId     // status
POST /api/marketplace/jobs/:runId/result           // agent upload (signed)
GET  /api/marketplace/me/usage                     // customer usage

4 · Implementation Phases (40h)

  1. Route scaffolding + auth (6h).
  2. Registration endpoint + Playbook upsert (6h).
  3. Job submission + run dispatch (8h).
  4. Result upload (signed token) (6h).
  5. Billing hook via PlatformMarkup (6h).
  6. Rate-limit + quota (4h).
  7. OpenAPI + smoke (4h).

5 · Risks & Mitigations

  • Result-upload abuse (huge payloads). Mitigation: signed URL + size cap + content-type allowlist.
  • Billing drift. Mitigation: idempotent PlatformMarkup keyed on runId; reconciliation cron daily.
  • Off-theme creep. Mitigation: documented as Apex enablement; reviewed by C4 owner.
