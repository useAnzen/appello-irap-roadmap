1 · Objective

agentc2/tests-e2e/ has 5 existing specs (navigation, agent-chat, downstream-flow, workflows-networks, nav-refactor) + the auth fixture; the appello repo has e2e screenshots of timesheet/approval flows. POC exists. This epic extends to 15 core construction workflows on top of the existing spec scaffolding — the workflows the Spider Testing Swarm needs to exercise to catch regressions in customer-critical paths.

2 · Acceptance Criteria

  • 15 spec files at appello/tests-e2e/flows/, covering: clock-in, clock-out, daily timesheet, weekly timesheet approval, T&M work order create, work-order line item add, work-order signature, change order create, change order approval, schedule view, schedule edit, photo upload, photo annotate, equipment add, material consumption.
  • Each spec inherits auth.fixture.ts + uses per-test tenant seed from create-test-tenant.ts.
  • Story tags applied per C1-0004 contract so trust-score wiring (C1-0024) can map specs to user stories.
  • Each spec emits a baseline trace + JSON metrics (re-uses C1-0010 baseline harness).
  • All 15 specs green on main; flake-rate < 1% measured over 10 nightly runs.
  • Mobile-Safari variants for the 5 most-used flows (clock-in, timesheet, photo-upload, schedule, signature).
  • Documentation at appello/tests-e2e/flows/README.md with per-flow seed-data expectations.

4 · Implementation Phases (60h)

  1. Per-flow seed-data design (8h).
  2. Specs 1-5 (clock + timesheet) (12h).
  3. Specs 6-10 (WO + change-order) (15h).
  4. Specs 11-15 (schedule + photo + equipment) (15h).
  5. Mobile-Safari variants for top 5 (6h).
  6. Flake stabilisation + nightly soak (4h).

5 · Risks & Mitigations

  • Seed-data setup per spec is slow. Mitigation: shared tenant snapshot + per-spec deltas.
  • Photo upload flake. Mitigation: mock file picker; verify upload via API not UI.
  • Signature drawing flake. Mitigation: programmatic mouse-move via Playwright trace recorder.
