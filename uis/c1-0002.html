1 · Objective

The Next.js route actions in appello-sdlc/apps/frontend/src/lib/actions/router/actions.ts already expose CRUD operations for work_packages_v2, work_items, stage_outputs, reviews, build_runs, test_runs, doc_artifacts and transition_logs. This epic completes that surface: 12 reviewable REST endpoints with Zod-validated bodies, a single error contract, idempotency keys on writes, and Vitest coverage that asserts the endpoint behavior the dashboard depends on. The endpoint set is the contract the dashboard and the hamburger-factory both consume — without the validation layer we have CRUD without guarantees.

2 · Acceptance Criteria

  • All 12 endpoints accept input via a shared z.object schema; reject 400 with { code, message, fieldErrors[] } on validation failure.
  • All endpoints return one of: 200 ok, 201 created, 400 invalid, 404 not_found, 409 conflict, 500 internal — no other shapes.
  • Write endpoints accept an Idempotency-Key header; duplicate keys within 24h return the original response.
  • Vitest suite covers each endpoint: happy path, validation failure, missing parent FK (404), idempotent replay (write only), and concurrent-update conflict (409).
  • Coverage gate ≥ 85% lines on the actions module enforced in CI before merge.
  • OpenAPI 3.1 spec emitted at /api/openapi.json from the Zod schemas (single source of truth).
  • Dashboard pages under apps/frontend/src/pages/* are unchanged in behavior — endpoint contracts are backward-compatible additions.

// apps/frontend/src/lib/actions/router/schemas.ts (new)
import { z } from 'zod';
export const CreateWorkPackageSchema = z.object({
  slug: z.string().min(1).max(120),
  title: z.string().min(1),
  parentWorkPackageId: z.string().uuid().optional(),
  uisId: z.string().optional(),
});
export type CreateWorkPackage = z.infer<typeof CreateWorkPackageSchema>;

The same schemas feed the OpenAPI generator (zod-to-openapi) so the spec is generated, not hand-written.

4 · Implementation Phases (35h)

  1. Shared error contract + idempotency middleware (5h) — author ApiError union type, response helper, idempotency-key store on the existing Postgres.
  2. Zod schemas for all 12 endpoints (8h) — request/response, shared types exported to dashboard.
  3. Wire schemas into the action router; align existing handlers (10h) — minimal diff, behavior preserved.
  4. Vitest coverage matrix (10h) — 5 cases per endpoint = 60 tests; CI coverage gate.
  5. OpenAPI emission + docs page (2h) — /api/openapi.json + /api/docs Swagger UI.

5 · Risks & Mitigations

  • Dashboard regression — Zod tightens what was previously permissive. Mitigation: shadow-mode flag for first week (log violations without rejecting).
  • Idempotency-key replay storage growth. Mitigation: 24h TTL + nightly cleanup job.
  • OpenAPI drift between schemas and runtime. Mitigation: CI contract test that hits each endpoint and validates response against the emitted spec.
