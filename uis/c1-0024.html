1 · Objective

agentc2/packages/agentc2/src/tools/test-runner-tools.ts dispatches the GitHub Actions stages (tsc, ESLint, unit, Playwright). playwright.config.ts is configured. The v4 hamburger factory is GitHub-first via wait-for-checks. This epic completes the loop: parse Playwright junit results from the GH artifact, score them per the C1-0004 contract, and write a Review.trustScore component so the autonomous merge gate has the actual E2E signal — not just compile success.

2 · Acceptance Criteria

  • test-runner-tools.ts.parsePlaywrightResults reads the junit XML from the GH artifact named playwright-junit.xml.
  • Per-spec results joined to hamburger.userStories[] via the @story:UIS-XX-S1 tag convention from C1-0004.
  • Score component playwrightScore emitted into Review.scoreBreakdown.playwright: 100 base, -30 per failing affected spec, -50 if a critical story is uncovered.
  • Review.trustScore recomputed via the existing weighted-sum formula in sdlc-reviewer; weights pinned in sdlc-reviewer.playbook.json.
  • If autoApprovePrBelow=medium and trust ≥ threshold AND no playwright fail on affected specs, autonomous squash-merge proceeds; else flag for review.
  • End-to-end smoke: 3 real hamburgers (1 all-pass, 1 playwright-fail, 1 unrelated-spec-fail) → trust score matches expected within ±5.
  • PR comment from sdlc-reviewer cites the playwright component score and which specs drove it.

3 · Architecture & Approach

Ingestion + scoring

// agentc2/packages/agentc2/src/tools/test-runner-tools.ts (extended)
export async function ingestPlaywright(prNumber: number, hamburger: Hamburger) {
  const xml = await downloadArtifact(prNumber, 'playwright-junit.xml');
  const results = parseJunit(xml);
  const affected = results.filter(r => r.tags.some(t => hamburger.userStories.includes(t)));
  const failed = affected.filter(r => r.status !== 'passed');
  const uncoveredCritical = hamburger.userStories.filter(s =>
    s.priority === 'P0' && !results.some(r => r.tags.includes(s))
  );
  return {
    playwrightScore: 100 - 30 * failed.length - (uncoveredCritical.length ? 50 : 0),
    failedSpecs: failed.map(f => f.title),
    uncoveredCritical,
  };
}

Review.trustScore breakdown

trustScore = w_tsc*tscScore + w_lint*lintScore + w_unit*unitScore + w_playwright*playwrightScore + w_scope*scopeScore

5 · Risks & Mitigations

  • Junit XML format changes after Playwright upgrade. Mitigation: Zod-parse the XML; fail loudly with version log.
  • Tag-convention drift. Mitigation: lint rule rejects specs without a story tag; nightly audit.
  • False auto-merge on critical-story gap. Mitigation: uncoveredCritical hard-blocks autonomous merge regardless of trust threshold.
