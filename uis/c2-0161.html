1 · Objective

appello-roadmap/plans/ is the prioritization surface; Fathom MCP + the fathom-meeting-summary workflow capture customer interviews; agentc2/apps/agent/src/app/pulse/[slug]/manage/page.tsx shows the AI-automation priority matrix. This epic wires them into one process: every customer interview → Fathom transcript → AI-extracted opportunity items → priority matrix cells in plans/AUTOMATION-MATRIX.md that feed C2-0157+ Spider coverage and C4 Apex toolbox planning.

2 · Acceptance Criteria

  • Documented process at appello-roadmap/processes/customer-interview-pipeline.md: who books, who runs, who reviews, where it lands.
  • Each interview tagged in Fathom with customer:<slug> and topic:automation-opportunity.
  • fathom-meeting-summary workflow extended with an extractor step: pulls candidate automation opportunities from the transcript as { painPoint, currentWorkaround, frequency, dollarImpact, automationDifficulty }.
  • Opportunities land in plans/AUTOMATION-MATRIX.md as rows in a 2D matrix (impact × difficulty).
  • Top-right quadrant (high-impact, low-difficulty) becomes input to next sprint planning (C1-0037).
  • Monthly review: matrix rows aged > 30 days without action revisited or archived.
  • 5 interviews executed in the first month to validate the pipeline.

3 · Architecture & Approach

Pipeline flow

Customer interview booked → call recorded in Fathom
  → fathom-meeting-summary workflow runs
    → summarizer agent (existing)
    → opportunity-extractor agent (new) outputs OpportunityRow[]
  → upsert into appello-roadmap/plans/AUTOMATION-MATRIX.md (PR opened)
  → Corey reviews + accepts
  → top-right items pulled into next sprint

Opportunity row schema

type OpportunityRow = {
  id: string;
  customer: string;
  painPoint: string;
  currentWorkaround: string;
  frequency: 'daily'|'weekly'|'monthly'|'one-off';
  dollarImpact: 'low'|'med'|'high';
  automationDifficulty: 'low'|'med'|'high';
  transcriptUrl: string;
  status: 'new'|'committed'|'building'|'shipped'|'declined';
};

4 · Implementation Phases (20h)

  1. Process doc + roles + cadence (3h).
  2. Fathom tagging convention + workflow extension (5h).
  3. Opportunity-extractor agent (5h).
  4. AUTOMATION-MATRIX.md template + PR-bot writer (4h).
  5. 5 interviews executed (3h).

5 · Risks & Mitigations

  • Extractor hallucinates opportunities. Mitigation: human review gate; declined rows tracked separately.
  • Customers ramble; transcripts noisy. Mitigation: agent prompt includes Appello-vocab grounding (APPELLO_VOCAB.md).
  • Matrix becomes wishlist that never ships. Mitigation: age-30d archival rule; sprint-planning enforced as the consumer.
