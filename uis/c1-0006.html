1 · Objective

The Next.js App Router shell for appello-sdlc exists with sidebar navigation across the Pipeline / Reviews / Runs surfaces (see screenshots 01–10 in the repo root). This epic finalises the information architecture and shared layouts so every future screen (Decomposer, Cost, Healer, etc.) gets a consistent shell for free: collapsible sidebar, breadcrumb-aware page header, shared empty/loading/error boundaries, and per-section deep-link patterns.

2 · Acceptance Criteria

  • Sidebar groups: Pipeline (work packages, runs, gates), Quality (reviews, healer, costs), Library (UIS specs, hamburgers, manifests), Admin (agents, models, integrations).
  • Each route has a page.tsx + layout.tsx co-located; shared (authenticated)/layout.tsx handles sidebar + header.
  • Breadcrumb component reads from a route-segment convention ((authenticated)/<section>/<resource>/[id]/page.tsx) — no manual breadcrumb config per page.
  • Shared EmptyState, LoadingState, ErrorBoundary components used by every page (no per-page reimplementation).
  • Sidebar state (collapsed/expanded) persisted in localStorage; respects prefers-reduced-motion.
  • Active-section highlight + keyboard navigation (↑/↓/Enter inside sidebar).
  • 100% Lighthouse accessibility score on shell + sample 3 pages.

3 · Architecture & Approach

Route layout

apps/frontend/src/app/
├── layout.tsx                          // root: fonts, theme, providers
├── (authenticated)/
│   ├── layout.tsx                      // sidebar + header
│   ├── pipeline/{page,layout}.tsx
│   ├── pipeline/[wp]/{page,layout}.tsx
│   ├── reviews/{page,layout}.tsx
│   ├── runs/{page,layout}.tsx
│   ├── library/uis/{page,layout}.tsx
│   ├── library/hamburgers/{page,layout}.tsx
│   └── admin/{agents,models,integrations}/{page,layout}.tsx
└── (public)/
    └── ... (sign-in, public UIS preview)

Shared components (new module _components/shell/)

  • Sidebar.tsx — collapsible, keyboard-nav, persisted state.
  • Breadcrumb.tsx — derives crumbs from route segments + page title context.
  • PageHeader.tsx — breadcrumb + title + slot for action buttons.
  • EmptyState.tsx / LoadingState.tsx / ErrorBoundary.tsx.

4 · Implementation Phases (35h)

  1. Route layout refactor — co-locate page/layout per segment (8h).
  2. Sidebar component + IA grouping (10h) — Storybook + a11y pass.
  3. Breadcrumb + PageHeader (5h).
  4. Shared boundaries (Empty/Loading/Error) + rollout across existing pages (8h).
  5. Keyboard nav + persisted state + Lighthouse pass (4h).

5 · Risks & Mitigations

  • Sidebar IA churn when new sections are added. Mitigation: IA owned by single registry file; adding section = one PR.
  • App-Router layout cascading bugs. Mitigation: Playwright snapshot test per page after refactor.
  • Accessibility regression. Mitigation: Lighthouse CI gate on shell + 3 sample pages.
