1 · Objective

agentc2/tests-e2e/ has Playwright baseline fixtures and the appello mono-repo has the real product flows that field crews use daily. This epic completes a structured baseline — task time, click count, error rate, drop-off — for the 5 highest-traffic flows (job-clock-in, daily-timesheet-submit, work-order-line-item-add, photo-upload, signature-capture). The baseline becomes the regression bar that any AI-generated UI change in C2/C4 has to clear before merge.

2 · Acceptance Criteria

  • Top-5 flows identified from production analytics (last 90 days, ranked by user-session count).
  • Each flow has: (a) Playwright spec under agentc2/tests-e2e/flows/, (b) baseline trace (.zip) captured at HEAD, (c) JSON baseline at tests-e2e/baselines/<flow>.json.
  • Baseline JSON shape: { flow, p50_ms, p95_ms, mean_clicks, error_rate, drop_off_step, sample_n, captured_at, commit_sha }.
  • Spec uses real production-shaped seed data (not synthetic empty state); seed script in tests-e2e/seeds/<flow>.ts.
  • Field-crew validation: 3 in-field sessions per flow recorded, baseline numbers within ±15% of observed real users.
  • Baselines refreshed on every release tag; CI fails if measured p95 regresses > 20% vs baseline.
  • Dashboard at /quality/baselines shows current vs baseline per flow.

3 · Architecture & Approach

Flow inventory (initial)

FlowWhy top-5
job-clock-in~3× per worker per day; first morning interaction
daily-timesheet-submitDaily, mandatory; gateway to payroll
work-order-line-item-addHighest-edit-velocity surface on T&M jobs
photo-uploadBandwidth-sensitive; offline path matters
signature-captureCustomer-facing; failure = unpaid invoice

Measurement methodology

Playwright trace + custom perfReporter emits the JSON above. Sample n = 30 runs per flow (10 cold cache, 10 warm, 10 throttled 3G). Error rate from in-spec assertions plus network 4xx/5xx.

Regression gate

// tests-e2e/util/regression-gate.ts
if (measured.p95_ms > baseline.p95_ms * 1.20) throw new Error(`Regression on ${flow}`);

4 · Implementation Phases (37h)

  1. Analytics query for top-5 traffic flows (4h).
  2. Seed scripts for each flow (8h).
  3. Playwright specs + traces (12h).
  4. Custom perf reporter + baselines JSON (4h).
  5. Field-crew validation sessions (5h).
  6. Regression gate + baseline dashboard (4h).

5 · Risks & Mitigations

  • Synthetic seeds ≠ field reality. Mitigation: field-crew validation step is mandatory acceptance.
  • Baselines decay as UI improves. Mitigation: baselines refreshed on every release tag; old ones archived.
  • Network throttling not representative of rural Canadian sites. Mitigation: include 3G + EDGE throttling profiles.
