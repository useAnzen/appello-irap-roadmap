1 · Objective

The work-package detail page in appello-sdlc/apps/frontend/ already renders enrichment output, partial loads, scope checks, and review state (screenshots 04, 06, 07, 08, 09, 10 in the repo root). This epic adds the work-item-level breakdown directly underneath the WP header: each WorkItem shows the AI plan (from StageOutput), the PR link, the trust score (from Review), and the live test runs. After this epic, the WP detail page tells the full story of one feature's autonomous build without leaving the page.

2 · Acceptance Criteria

  • WP detail page lists all child WorkItems in dependency order under the existing WP header.
  • Each work-item row shows: title, stage, latest StageOutput excerpt, PR link (if open), Review.trustScore, last BuildRun.status, last TestRun.summary.
  • Click → expandable panel: full StageOutput log, full Review.comments, per-check breakdown from BuildRun.checks.
  • Live updates via the SSE channel from C1-0005 (no separate stream).
  • Trust score visually encoded: green ≥80, amber 60–79, red <60; tooltip shows score breakdown (tsc / eslint / playwright / scope).
  • PR link includes branch name + open/merged/closed badge; deep-links to GitHub.
  • Empty state: WP with zero work items shows decomposer status, not an error.

3 · Architecture & Approach

Data fetching

// apps/frontend/src/app/pipeline/[wp]/page.tsx (extended)
const wp = await db.workPackage.findUnique({
  where: { id: params.wp },
  include: {
    workItems: {
      orderBy: { sequenceNumber: 'asc' },
      include: {
        stageOutputs: { orderBy: { createdAt: 'desc' }, take: 1 },
        review: true,
        buildRuns: { orderBy: { createdAt: 'desc' }, take: 1 },
        testRuns:  { orderBy: { createdAt: 'desc' }, take: 1 },
      }
    }
  }
});

Component layout

  • WorkItemRow.tsx — collapsed row (title, stage pill, PR badge, trust score).
  • WorkItemPanel.tsx — expanded body (full logs, review comments, per-check table).
  • TrustScoreBadge.tsx — color rule + tooltip with score components.

4 · Implementation Phases (45h)

  1. WorkItemRow + collapsed UI (8h).
  2. Expandable panel with full logs + per-check table (10h).
  3. TrustScoreBadge + breakdown tooltip (4h).
  4. PR badge component + GH state polling (5h).
  5. Live SSE wiring (re-using C1-0005 stream) (6h).
  6. Empty states + error boundaries (4h).
  7. Playwright + Storybook coverage of the 6 work-item states (8h).

5 · Risks & Mitigations

  • Heavy DB join on large WPs. Mitigation: paginate work items (20/page); lazy-load expanded panels.
  • PR badge flakes if GH rate-limits. Mitigation: cache PR state for 60s; degrade gracefully to last-known.
  • Log tail UX becomes noisy. Mitigation: filter chips for level/stage; collapse-by-default.
