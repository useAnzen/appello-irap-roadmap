1 · Objective

Migrate the ATLAS knowledge base — 729+ Fathom call transcripts plus onboarding notes and internal docs currently indexed by an N8n + Google Drive pipeline writing into a Supabase vector store — onto the AgentC2 RAG stack (packages/agentc2/src/rag/pipeline.ts + packages/agentc2/src/vector.ts) backed by Postgres pgvector with the RagDocument / RagChunk models. The cutover replaces the externally-hosted N8n MCP endpoint (useappello.app.n8n.cloud/mcp/...) with the in-repo ragQueryTool and brings tenancy, idempotency, embedding-model control, and rollback into version-controlled code.

This epic is the planning + cutover artifact. The destination architecture is already shipped in agentc2; the migration of the actual 729+ transcript corpus is the gap.

2 · Acceptance Criteria

  1. All 729+ Fathom transcripts present in the legacy N8n/Drive pipeline are re-ingested into rag_documents with organization_id populated and chunks mirrored into rag_chunk for hybrid search.
  2. A golden set of ≥ 25 representative ATLAS queries returns top-K results whose normalized text-similarity vs. legacy top-K is ≥ 0.90 average (per §12 test plan).
  3. Re-running the migration script over the same corpus is idempotent: zero duplicate vectorIds, zero net new RagChunk rows on second pass.
  4. The legacy N8n MCP URL useappello.app.n8n.cloud/mcp/dfbad0dd-acf3-4796-ab7a-87fdd03f51a8/sse is removed from ~/.cursor/mcp.json (team-wide rollout) and the AgentC2 ragQueryTool is wired as the replacement; legacy N8n workflow is paused (not deleted) and scheduled for full decommission 14 days after parallel-run sign-off.
  5. End-to-end p95 query latency < 500 ms against the production Supabase pgvector index for top-K = 5, minScore = 0.5, organization-scoped queries.
  6. Re-embedding cost for the full corpus is captured, fits within an approved IRAP infra budget line, and includes a per-document cost tag in RagDocument.metadata.
  7. Tenant isolation is verified: a probe query with a synthetic organizationId returns zero ATLAS chunks (enforces the production guard in queryRag).
  8. A rollback runbook is committed under agentc2/docs/atlas-migration/ with the steps to re-activate the legacy N8n MCP within ≤ 30 minutes for the parallel-run window.
  9. The N8n workflow definition (currently only in n8n.cloud UI) is exported to JSON and committed to the repo before cutover, so the migration is reproducible and the legacy behavior is auditable.
  10. An NRC-shippable evidence packet is produced: this UIS + the migration script + the golden-set diff report + the cost ledger + the rollback runbook.

3 · Current state — the legacy ATLAS pipeline

ATLAS today is an externally-hosted N8n workflow on useappello.app.n8n.cloud exposed to Cursor and other clients as an SSE MCP endpoint. The setup guide is committed in the repo at agentc2/docs/ATLAS-MCP-SETUP.md and documents the public-facing surface only — a single tool Query_ATLAS that accepts a free-text query and returns pageContent / metadata / score per hit.

3.1 What we know (from the committed setup doc)

  • MCP transport: SSE at https://useappello.app.n8n.cloud/mcp/dfbad0dd-acf3-4796-ab7a-87fdd03f51a8/sse, no auth, single tool Query_ATLAS(query: string).
  • Behavior: semantic search; scores typically 0.3–0.7; metadata includes source file, document title, and line numbers — implying chunk-level granularity and a Google-Drive-rooted source corpus.
  • Corpus shape (per the master sheet activity row): 729+ Fathom call transcripts plus onboarding calls, internal docs, and operational context.

  • Chunker: presumed default N8n / LangChain recursive character splitter (~1000 chars, ~200 overlap). Actual chunkSize / chunkOverlap / separators must be read off the workflow node.
  • Embedder: presumed OpenAI text-embedding-3-small (1536-dim) given the destination is also 1536-dim — but could be ada-002 in the legacy flow; if so, all vectors must be regenerated (they will be regardless, see §9).
  • Vector store: presumed Supabase Vector node writing to a project we control. Schema, index name, and metric (cosine vs. ip vs. l2) must be confirmed.
  • Tenant model: single-tenant — no organization_id column observed in the public MCP responses; chunks are returned to any caller. The destination enforces per-org isolation (see §6 + §7).
  • Refresh cadence: presumed cron-trigger on a fixed interval; actual schedule and de-dupe strategy (modified-time watermarks vs. full re-scan) need confirmation.

6 · Chunker + embedding implementation (real code)

6.1 Embedder — OpenAI text-embedding-3-small @ 1536-dim

Source: packages/agentc2/src/rag/pipeline.ts:10-17

async function getEmbedder(organizationId?: string) {
    const apiKey = await getOrgApiKey("openai", organizationId);
    if (!apiKey) {
        throw new Error("OpenAI API key not configured. Add it via Settings > Integrations.");
    }
    const openai = createOpenAI({ apiKey });
    return openai.embedding("text-embedding-3-small");
}

6.2 Chunker — MDocument.chunk() @ recursive 512 / 50

Source: packages/agentc2/src/rag/pipeline.ts:46-66

export async function chunkDocument(
    doc: MDocument,
    options: ChunkOptions = {}
): Promise<Array<{ text: string; metadata: Record<string, any> }>> {
    const { strategy = "recursive", maxSize = 512, overlap = 50 } = options;

    const chunks = await doc.chunk({ strategy, maxSize, overlap });

    return chunks.map((chunk, index) => ({
        text: chunk.text,
        metadata: { chunkIndex: index, charCount: chunk.text.length, ...chunk.metadata }
    }));
}

6.3 Index init — Supabase pgvector, 1536-dim, cosine

Source: packages/agentc2/src/rag/pipeline.ts:71-82

const RAG_INDEX_NAME = "rag_documents";

export async function initializeRagIndex(): Promise<void> {
    const indexes = await vector.listIndexes();
    if (!indexes.includes(RAG_INDEX_NAME)) {
        await vector.createIndex({
            indexName: RAG_INDEX_NAME,
            dimension: 1536, // OpenAI text-embedding-3-small dimension
            metric: "cosine"
        });
    }
}

6.4 Ingest — embed → upsert → backfill organization_id → mirror to RagChunk

Source: packages/agentc2/src/rag/pipeline.ts:120-180 (abridged)

const embeddingModel = await getEmbedder(organizationId);
const { embeddings } = await embedMany({
    model: embeddingModel,
    values: chunks.map((c) => c.text)
});

const vectorIds = chunks.map((_, i) => `${documentId}_chunk_${i}`);

await initializeRagIndex();
await vector.upsert({
    indexName: RAG_INDEX_NAME,
    vectors: embeddings,
    metadata,
    ids: vectorIds
});

// Backfill the dedicated org column for DB-level filters / RLS
await prisma.$executeRawUnsafe(
    `UPDATE rag_documents SET organization_id = $1
     WHERE metadata->>'organizationId' = $1 AND organization_id IS NULL`,
    organizationId
);

// Dual-write chunks for full-text search (RRF input)
await prisma.ragChunk.createMany({
    data: chunks.map((chunk, i) => ({
        id: `${documentId}_chunk_${i}`, documentId,
        organizationId: organizationId || "",
        chunkIndex: i, text: chunk.text, sourceName: sourceName || documentId,
        metadata: chunk.metadata
    })),
    skipDuplicates: true
});

6.6 Tool surface — what Cursor / agents call

Source: packages/agentc2/src/tools/rag-tools.ts:9-38

export const ragQueryTool = createTool({
    id: "rag-query",
    description: "Query the RAG index without generating a response.",
    inputSchema: z.object({
        query: z.string(),
        topK: z.number().optional(),
        minScore: z.number().optional(),
        organizationId: z.string().optional()
            .describe("Organization ID for tenant-scoped queries (auto-injected)")
    }),
    outputSchema: baseOutputSchema,
    execute: async ({ query, topK, minScore, organizationId }) => {
        const results = await queryRag(query, { organizationId, topK, minScore });
        return { success: true, results: results.map(r => ({
            text: r.text, score: r.score, metadata: JSON.parse(JSON.stringify(r.metadata))
        })), resultCount: results.length };
    }
});

7.1 Rollback strategy

≤ 30 minute rollback window during parallel run.
  1. Re-enable the legacy N8n cron trigger from n8n.cloud UI (single toggle).
  2. Revert the ~/.cursor/mcp.json doc change; broadcast the prior endpoint URL to the team via Slack.
  3. Optionally UPDATE rag_documents SET organization_id = NULL WHERE metadata->>'migrationBatch' = 'c3-0231-v1' to logically detach migrated chunks from the ATLAS org without dropping data (preserves diff history for post-mortem).
  4. File a rollback note under agentc2/docs/atlas-migration/rollbacks/<date>.md capturing trigger, blast radius, and remediation.
The rollback is reversible at any point: re-running the back-fill script re-attaches the chunks (idempotent UPDATE).

8 · Cost estimate (re-embedding the full ATLAS corpus)

OpenAI text-embedding-3-small is billed at $0.020 per 1M input tokens (2026 pricing). The dominant cost is the one-shot back-fill; steady-state incremental embeds are negligible at the corpus's growth rate.

Worst-case (10× the token estimate, full reprocessing two times during parallel run, model price doubles) still lands < $10. The infra-cost line item for this epic is dominated by Corey's engineering time, not by OpenAI usage.

Vendor pricing drift. If OpenAI deprecates text-embedding-3-small or repricing pushes per-1M token cost above $0.20, the budget remains under $100 for the corpus. The risk is not cost — it is having to re-embed against a successor model whose vectors are not interoperable with the existing 1536-dim index (forces a full repeat of Phase 3).
