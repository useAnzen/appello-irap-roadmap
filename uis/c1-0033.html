1 · Objective

appello-roadmap/hamburger-manifest.json + hamburger-manifest-1.34.json are the golden references; agentc2/scripts/seed-sdlc-playbook.ts establishes the seed/compare pattern. This epic builds the automated scorer that diffs a new decomposer output against a golden manifest and produces a quantitative quality score: hamburger-count parity, dependency-graph isomorphism, change-surface coverage, AC recall, sequencing similarity.

2 · Acceptance Criteria

  • tools/decomposer-scorer.ts takes (actualOutput, goldenManifest) → { score: 0-100, breakdown: {...}, diffs: {...} }.
  • Component: count parity — 25 points; perfect = same hamburger count; ±1 = -5; ±2 = -10; >±2 = -25.
  • Component: DAG isomorphism — 25 points; relaxed labeling allowed; predecessor/successor topology must match.
  • Component: change-surface coverage — 25 points; union of actual files ⊇ 90% of golden files = full credit; linear penalty below.
  • Component: AC recall — 25 points; string-similarity (TF-IDF cosine) + LLM judge for semantic-equivalence on near-misses.
  • CLI: bun run decomposer-scorer <actual.json> <golden.json>; outputs JSON + markdown summary.
  • Integrates with C1-0032 test harness as the scoring backend.
  • Calibrated against 5 hand-scored manifest pairs; calibration sheet checked into tools/scorer-calibration.md.

3 · Architecture & Approach

Scorer breakdown

// tools/decomposer-scorer.ts
export async function scoreDecomposition(actual: Manifest, golden: Manifest) {
  const countParity   = scoreCountParity(actual.hamburgers.length, golden.hamburgers.length);
  const dagIso        = scoreDagIsomorphism(actual.dependencyGraph, golden.dependencyGraph);
  const surfaceCov    = scoreChangeSurface(
    flattenSurface(actual.hamburgers),
    flattenSurface(golden.hamburgers)
  );
  const acRecall      = await scoreAcRecall(actual.hamburgers, golden.hamburgers, { llmJudge: true });

  return {
    score: countParity + dagIso + surfaceCov + acRecall,
    breakdown: { countParity, dagIso, surfaceCov, acRecall },
    diffs: {
      missingFiles: missing(golden, actual),
      addedFiles:   missing(actual, golden),
      acGaps:       acGaps(actual, golden),
    }
  };
}

DAG isomorphism — relaxed

Slugs differ between actual and golden; isomorphism is computed on edge-shape only (canonicalized: sort children by degree). Two graphs are isomorphic iff there's a bijection preserving edges.

5 · Risks & Mitigations

  • DAG isomorphism NP-hard worst case. Mitigation: heuristic acceptable; graphs are small (≤30 nodes); fallback to edit-distance if isomorphism check exceeds 5s.
  • AC recall LLM-judge expensive. Mitigation: LLM only on TF-IDF near-misses (similarity 0.5-0.85); save 80% of calls.
  • Scorer drift after embedding-model upgrade. Mitigation: version-pin embedding model in calibration sheet.
