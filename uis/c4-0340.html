1 · Objective

Migrations folder + Prisma is the migration framework already in use across the platform (agentc2/packages/database/prisma/migrations/ and appello-sdlc/application-mono-repo/apps/api/prisma/migrations/202407/). This epic extends those patterns for the C4 Apex schema work: ApexToolbox, ApexSkill, ApexRun, ApexBillingEvent — the foundational data model for the customer-facing Apex product.

2 · Acceptance Criteria

  • ApexToolbox: { slug, name, ownerCustomerId?, isPublic, skills }.
  • ApexSkill: { slug, toolboxId, name, description, parameterSchema(Json), implementationRef }.
  • ApexRun: { id, skillId, customerId, input, output, status, costUsd, latencyMs, startedAt, completedAt? }.
  • ApexBillingEvent: { id, runId, customerId, amountUsd, markupPct, settledAt? } — links to existing PlatformMarkup.
  • Indexes: ApexRun(customerId, startedAt DESC), ApexRun(skillId, status).
  • Migration additive only; backfill not needed (new product).
  • RLS policies: customer can only see own runs + public toolboxes.
  • Prisma client regenerated; types exported to the broader agentc2 package.

3 · Architecture & Approach

Schema

model ApexToolbox {
  id              String @id @default(cuid())
  slug            String @unique
  name            String
  ownerCustomerId String?
  isPublic        Boolean @default(false)
  skills          ApexSkill[]
  createdAt       DateTime @default(now())
}

model ApexSkill {
  id                 String @id @default(cuid())
  slug               String @unique
  toolbox            ApexToolbox @relation(fields: [toolboxId], references: [id])
  toolboxId          String
  name               String
  description        String
  parameterSchema    Json
  implementationRef  String  // e.g. agent slug or MCP tool ref
  runs               ApexRun[]
}

model ApexRun {
  id           String @id @default(cuid())
  skill        ApexSkill @relation(fields: [skillId], references: [id])
  skillId      String
  customerId   String
  input        Json
  output       Json?
  status       ApexRunStatus
  costUsd      Decimal @db.Decimal(10, 4)
  latencyMs    Int?
  startedAt    DateTime @default(now())
  completedAt  DateTime?
  billingEvent ApexBillingEvent?
  @@index([customerId, startedAt(sort: Desc)])
  @@index([skillId, status])
}

model ApexBillingEvent {
  id          String @id @default(cuid())
  run         ApexRun @relation(fields: [runId], references: [id])
  runId       String  @unique
  customerId  String
  amountUsd   Decimal @db.Decimal(10, 4)
  markupPct   Decimal @db.Decimal(5, 2)
  settledAt   DateTime?
}

enum ApexRunStatus { pending running success failure cancelled }

5 · Risks & Mitigations

  • Schema lock-in pre-customer-feedback. Mitigation: minimal scope; additive iterations expected.
  • RLS misconfiguration leaks runs across customers. Mitigation: RLS unit test per policy.
  • Cost field precision. Mitigation: Decimal(10,4) for sub-cent precision.
