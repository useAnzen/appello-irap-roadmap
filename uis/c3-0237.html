1 · Objective

agentc2 cost-tracking tables (AgentModelCostDaily, CostEvent) + the budget package + AgentEvaluation outcomes are the data surface for the production-model selection decision. This epic produces the decision document: which model is the default for each SDLC agent role (decomposer, planner, auditor, reviewer, healer), with cost projection at expected volume and the eval-numbers that justify the choice.

2 · Acceptance Criteria

  • Decision document at agentc2/reports/model-selection-2026Q2.md per agent role.
  • Each section: chosen model, evidence (C3-0230 rubric scores), cost projection at 1× / 5× / 25× current volume, rollback plan.
  • Cost projection sources: per-task cost from AgentEvaluation, expected volume from AgentRun + planned hamburger throughput.
  • Per-role recommendation includes fallback model if vendor outage.
  • Sign-off: Corey + Christopher + Ian; recorded as ADR (C2-0159 framework).
  • Default model wired into agentc2/packages/agentc2/src/agents/<agent>.ts; rollback = single PR.
  • Quarterly review baked into C2-0159 cadence.

4 · Implementation Phases (25h)

  1. Pull C3-0230 rubric results + per-agent slice (4h).
  2. Cost projection model 1×/5×/25× (6h).
  3. Per-role recommendation + rationale (6h).
  4. Sign-off review with Corey/Christopher/Ian (3h).
  5. Wire defaults + ADR commit (4h).
  6. Cost-monitor dashboard tie-in (2h).

5 · Risks & Mitigations

  • Vendor price change post-decision. Mitigation: projection includes ±50% sensitivity; reviewed quarterly.
  • Per-role lock-in. Mitigation: abstraction layer in agent.ts; model is a config not hard-code.
  • Sign-off delay. Mitigation: single 1-hour review meeting; written async pre-read.
