1 · Objective

agentc2/Dockerfile + agentc2/DEPLOY.md describe the production stack (DO compose + Caddy); agentc2/apps/agent/src/lib/inngest.ts is the worker layer; AgentModelCostDaily is the cost telemetry. This epic introduces dedicated model-serving endpoints with auto-scaling layered on top: per-model worker pools (Sonnet pool, GPT-4o pool, embedding pool) sized by recent demand, with circuit breakers per vendor and graceful fallback to the C3-0237 fallback model on outage.

2 · Acceptance Criteria

  • Per-model worker pools: worker-sonnet, worker-gpt4o, worker-embedding; each scaled independently.
  • Auto-scaling: pool size adjusted every 5 min based on prior 15-min queue depth + p95 latency target.
  • Circuit breaker per vendor: 5 consecutive 5xx → open for 60s; half-open probe; closed on success.
  • Open circuit → traffic routed to fallback model from C3-0237 ADR; downgrade logged in AgentInvocation.downgradeReason.
  • Cost guardrail: per-pool daily spend cap; soft-stop at 90% with alert (C1-0025), hard-stop at 100%.
  • Endpoints behind internal-only DNS; rate limit per agent slug.
  • Observability: per-pool Prometheus-compatible metrics; Grafana dashboard.
  • Soak: 200 concurrent invocations × 10 min; no breaker trips on healthy vendors.

3 · Architecture & Approach

Pool topology

+-----------+    +------------------+    +----------+
|  agents   | -> |   worker pool    | -> |  vendor  |
+-----------+    +------------------+    +----------+
   inngest         per-model autoscale       claude
                                              gpt-4o
                                              gemini

Circuit breaker

// per-vendor state machine
state: CLOSED | OPEN | HALF_OPEN
CLOSED   --(5 consecutive 5xx)--> OPEN
OPEN     --(60s)-->               HALF_OPEN
HALF_OPEN --(probe success)-->     CLOSED
HALF_OPEN --(probe fail)-->        OPEN

4 · Implementation Phases (50h)

  2. Autoscaler algorithm + cron (10h).

  4. Fallback model routing (6h).
  5. Cost guardrails + alert hookup (5h).
  6. Internal DNS + rate limits (4h).
  7. Observability + Grafana (6h).
  8. Soak test + tuning (6h).

5 · Risks & Mitigations

  • Autoscaler thrashing. Mitigation: cooldown 5 min; min-pool floor.
  • Vendor outage cascade. Mitigation: circuit breaker + fallback; both models monitored independently.
  • Cost runaway from autoscale. Mitigation: hard daily cap; pool cannot grow past max-size config.
