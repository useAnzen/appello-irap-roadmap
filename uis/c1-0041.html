1 · Objective

appello-sdlc/packages/database/ already uses Prisma's built-in prisma migrate toolchain. This epic packages it as a CLI matching the convention from appello-xrm-prisma-migrations: idempotent SQL, auto-baselining behavior, safe-rollout discipline. After this epic every schema change across the monorepo family is run through the same CLI with the same safety guards.

2 · Acceptance Criteria

  • CLI binary appello-migrate exposed via bun appello-migrate <cmd>.
  • Commands: plan (generate migration), up (apply to local), deploy (apply to env), down (rollback 1), validate (check idempotency + safety rules).
  • Safety rules from xrm convention: no DROP COLUMN without companion idempotent SQL; no destructive raw SQL outside @audit-allow-destructive annotation; auto-baseline new envs.
  • plan opens generated SQL in $EDITOR before commit; refuses if SQL contains a rule violation.
  • deploy in CI uses prisma migrate deploy under the hood; never migrate dev.
  • Migration files follow naming: YYYYMMDDHHMMSS_<snake_case_desc>/migration.sql.
  • validate idempotency: simulate applying migration twice; pass if second apply is no-op.
  • Documentation in packages/database/MIGRATIONS.md describes the new convention + escape hatches.

Idempotency simulator

// packages/database/cli/validate.ts
const tempDb = await spawnTempPostgres();
await runMigration(migration, tempDb);
const snap1 = await dumpSchema(tempDb);
await runMigration(migration, tempDb);  // re-apply
const snap2 = await dumpSchema(tempDb);
if (!deepEqual(snap1, snap2)) throw new Error('migration not idempotent');

4 · Implementation Phases (40h)

  1. CLI scaffolding (commander.js / clipanion) (4h).
  2. plan/up/deploy/down commands (12h).
  3. Safety-rule linter (idempotency + destructive checks) (10h).
  4. Idempotency simulator with tempPostgres (8h).
  5. CI integration + migrate deploy wrapper (4h).
  6. MIGRATIONS.md + examples (2h).

5 · Risks & Mitigations

  • Down-migrations not always derivable. Mitigation: hand-written down.sql required for non-additive migrations; lint blocks merge if missing.
  • Validate slow with tempPostgres. Mitigation: shared container per CI worker; skipped for docs-only changes.
  • Convention divergence between XRM and SDLC. Mitigation: CLI lives in shared @appello/migrate package; one source of truth.
