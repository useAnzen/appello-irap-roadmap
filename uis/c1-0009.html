1 · Objective

agentc2/packages/agentc2/src/tools/rag-tools.ts already writes documents into RagDocument + RagChunk with pgvector embeddings; agentc2/docs/ATLAS-MCP-SETUP.md is the deployment guide. This epic produces the labelled corpus: every historical Jira/GitHub bug ticket and feature request in the Appello backlog gets normalised, classified, and ingested via rag-tools so the uis-decomposer agent (C1-0029) can retrieve precedent-aware context ("how did we slice last time we touched the rates engine?"). Important: this is retrieval context for the agent at runtime — not a fine-tuning dataset.

2 · Acceptance Criteria

  • Corpus ≥ 500 labelled examples spanning bugs + features across the last 24 months from Jira + GitHub Issues.
  • Each example normalised to: { id, source, title, body, resolution, labels: { type, severity, area, complexity }, mergedPrSha?, hamburgerSlug? }.
  • Classification labels assigned via a deterministic rules-first pass + LLM tie-breaker (logged); precision ≥ 0.85 on a 50-row hand-labelled holdout.
  • Ingested through rag-tools.upsertDocument with collection='backlog-precedent'; chunked at 800 tokens with 100-token overlap.
  • Embedding model documented (currently text-embedding-3-small); rebuild script versioned in repo.
  • Decomposer agent prompt augmented with a retrieval tool call: retrievePrecedent(scope) → top-5 chunks.
  • A/B eval against the C1-0034 golden manifests shows decomposer success-rate improvement ≥ +10 absolute points (currently 22% baseline).

3 · Architecture & Approach

Ingestion pipeline

Jira REST / GH API
    ↓ (fetch_backlog.ts — paginated, idempotent)
normalize_corpus.ts  →  rules_classifier.ts  →  llm_classifier.ts (tie-break)
    ↓
rag-tools.upsertDocument({ collection: 'backlog-precedent', body, metadata })
    ↓
RagDocument + RagChunk (pgvector)

Retrieval at decomposer runtime

// agents/uis-decomposer prompt (excerpt)
Tool: retrievePrecedent({ area: techSpec.area, complexity: 'high' })
→ top-5 chunks of past slicings for similar scope.
Use them to inform hamburger boundaries; do NOT copy literal slugs.

Label taxonomy lives in agentc2/packages/agentc2/src/lib/labels.ts — single source for classifier + decomposer.

5 · Risks & Mitigations

  • Mislabelled corpus poisons retrieval. Mitigation: 0.85 precision gate on holdout before ingest; quarterly re-label audit.
  • PII in historical tickets. Mitigation: normaliser strips email/phone via regex + Presidio.
  • Embedding-model upgrade requires full reindex. Mitigation: rebuild script + version field on collection.
