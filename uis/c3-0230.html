1 · Objective

agentc2/packages/agentc2/src/scorers/ + agent-quality-tools.ts run model evaluations; agentc2/reports/mcp-expert-performance.md is the eval-report-format precedent; AgentEvaluation persists outcomes. This epic runs a structured rubric across GPT-4o, Claude Sonnet+Opus, Gemini 1.5/2, and Llama 3 on the agentc2 scorer harness with results persisted in AgentEvaluation. Outcome = the cost/quality table that drives the C3-0237 model-selection decision.

2 · Acceptance Criteria

  • Rubric covers 5 dimensions: instruction-following, code-generation (TypeScript + Prisma), structured-output (JSON schema adherence), reasoning depth, latency.
  • Test corpus = 30 SDLC-realistic prompts spanning UIS-generator, decomposer, planner, auditor, reviewer tasks (sourced from C1-0019 construction scenarios).
  • Each model × prompt = 1 AgentEvaluation row: { model, version, promptId, score, costUsd, latencyMs, outputJson }.
  • Scorer harness emits per-dimension score (0-100) plus aggregate.
  • Report at agentc2/reports/model-rubric-2026Q2.md following the mcp-expert-performance.md precedent.
  • Cost per task computed from AgentModelCostDaily.
  • Heatmap visualisation: model vs. dimension; cost-quality scatter plot.
  • Findings include a clear recommendation per agent role (which model to default).

3 · Architecture & Approach

Eval pipeline

// agentc2/scripts/run-model-rubric.ts
for (const model of MODELS) {
  for (const prompt of CORPUS) {
    const start = Date.now();
    const out = await callModel(model, prompt.text);
    const latencyMs = Date.now() - start;
    const score = await scorerHarness.score(prompt.expectations, out);
    const costUsd = costFromTokens(model, prompt.text.length, out.length);
    await db.agentEvaluation.create({ data: {
      model: model.id, version: model.version, promptId: prompt.id,
      score, costUsd, latencyMs, outputJson: out
    }});
  }
}

5 · Risks & Mitigations

  • Rubric weights subjective. Mitigation: 2 sets of weights reported (engineering bias vs cost bias).
  • Model variability run-to-run. Mitigation: 3 samples per (model, prompt); report mean ± stddev.
  • Cost numbers stale post-vendor-price-change. Mitigation: cost re-computed at report time; vendor-price snapshot SHA in report.
