1 · Objective

agentc2/packages/agentc2/src/mcp/api-clients/jira.ts is the Jira MCP client; appello/plan/JIRA_OUTSTANDING_TICKETS.md is the manually-tracked outstanding list; appello-roadmap/ holds the higher-order backlog. Today these three surfaces are independent. This epic merges them into one operational queue with daily 15-min standups against it, so triage stops being a daily search across three places.

2 · Acceptance Criteria

  • Single backlog page at /pipeline/backlog ingests rows from Jira (via MCP), JIRA_OUTSTANDING_TICKETS.md (parsed), and appello-roadmap/plans/INDEX.md P0/P1 items.
  • Each row tagged with source, priority, age_days, assignee, linked_hamburger?.
  • Standup cadence: 15 min daily @ 9:15 ET; agenda = yesterday's completes + today's plan + blockers; auto-populated from the backlog page.
  • Blockers auto-flagged: age_days > 7 on a P0, linked_hamburger.state = failed, review.status = PENDING > 4h (see C1-0036).
  • Standup notes appended to appello-roadmap/standups/YYYY-MM-DD.md; export to Slack #sdlc-team.
  • JIRA_OUTSTANDING_TICKETS.md deprecated once parity is reached; replaced by a generated view from the backlog page.
  • Backlog refreshes every 5 min from Jira MCP (rate-limit aware).

3 · Architecture & Approach

Backlog ingestion

// apps/agent/src/lib/backlog/ingest.ts (cron */5 min)
const jiraTickets = await jiraMcp.searchIssues({ jql: 'project = APP AND status != Done' });
const outstanding = parseMarkdownList(await fs.readFile('appello/plan/JIRA_OUTSTANDING_TICKETS.md'));
const plans = parsePlansIndex(await fs.readFile('appello-roadmap/plans/INDEX.md'));

const merged = dedupBy(
  [...mapJira(jiraTickets), ...mapOutstanding(outstanding), ...mapPlans(plans)],
  r => `${r.source}:${r.externalId}`
);
await db.backlogRow.upsertMany(merged);

4 · Implementation Phases (15h)

  1. Backlog ingestion script + cron (4h).
  2. Backlog page + filters/sort (5h).
  3. Standup auto-agenda + Slack export (3h).
  4. Blocker detection rules (2h).
  5. Deprecation of JIRA_OUTSTANDING_TICKETS.md (1h).

5 · Risks & Mitigations

  • Jira MCP rate-limit hit. Mitigation: incremental updatedAfter query; backoff on 429.
  • Dedup misses near-duplicates. Mitigation: manual merge UI for ambiguous matches.
  • Standups become note-recital. Mitigation: 15-min hard cap; blocker rows surfaced first.
