1 · Objective

The canonical NH1 onboarding surface for the stack itself (distinct from the SDLC pipeline guided tour in C1-0018): agentc2/CLAUDE.md, agentc2/DEPLOY.md, the Bun + Turborepo + Prisma stack in appello-sdlc/packages/, and agentc2/docs/sdlc/03-hamburger-factory.md. This epic produces the codebase onboarding doc + 2 paired-coding sessions that get NH1 from "clone the repo" to "merged a real hamburger" in 2 weeks.

2 · Acceptance Criteria

  • Doc agentc2/docs/onboarding/NH1-codebase-tour.md covers: monorepo layout, Bun workspace conventions, Turborepo pipeline graph, Prisma schema + migrations workflow, Inngest topology.
  • Each section has a 15-min reading + a 30-min hands-on exercise (e.g. "add a Prisma field, generate, write a migration, roll forward and back").
  • Hamburger-factory deep dive: walk one historical successful hamburger end-to-end with NH1 reading every PR comment, every trust score, every CI run.
  • Paired session 1 (week 1): NH1 + Ian implement a tiny P2 hamburger together, narrating the keystrokes.
  • Paired session 2 (week 2): NH1 implements a P2 hamburger solo with Ian on call; goal = merged PR without external help.
  • Onboarding signed off by NH1 + Ian when paired session 2 succeeds.
  • Time-on-doc tracked; if > 40h hands-on, the doc itself gets refactored.

  1. Onboarding doc authoring (12h).

5 · Risks & Mitigations

  • Overlap with C1-0018 SDLC tour. Mitigation: C1-0018 = pipeline tour; C1-0020 = stack tour; cross-link explicitly.
  • Stack changes between authoring and NH1 start (Bun/Turbo bumps). Mitigation: Ian re-runs the day-1 exercise week-of.
  • NH1 hire date slip. Mitigation: doc is useful even without NH1 (becomes the public onboarding ref).
