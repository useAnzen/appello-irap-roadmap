1 · Objective

agentc2/vitest.config.ts + APPELLO-XRM/scripts/vitest-testcontainers-global-setup.ts are the established pattern for isolated DB-backed Vitest. This epic writes the test suites for the migrate CLI (C1-0041) and the agent-status hooks (C1-0043) using that pattern, so both new surfaces ship with the same testing discipline as the rest of the monorepo.

2 · Acceptance Criteria

  • Migrate CLI suite: packages/migrate/__tests__/cli.test.ts covers plan/up/deploy/down/validate against a per-test testcontainers Postgres.
  • Validate-idempotency suite: applies each fixture migration twice, asserts no-op on second apply.
  • Hook suite: apps/frontend/src/hooks/__tests__/agent-runs.test.ts uses React-Testing-Library + msw to mock SSE.
  • Hook suite covers: initial load, SSE update merge, optimistic mutation rollback on error, cache invalidation.
  • Coverage gate ≥ 85% lines on the migrate CLI + 80% on hooks.
  • All tests obey assert-safe-db guards (no destructive against shared dev DB).
  • CI matrix: Postgres 15 + 16 to catch version-specific regressions.
  • Total wall-clock < 90s on dev hardware (warm container).

3 · Architecture & Approach

Testcontainers setup

// packages/migrate/__tests__/setup.ts
import { PostgreSqlContainer } from '@testcontainers/postgresql';

export async function spawnTestDb() {
  const c = await new PostgreSqlContainer('postgres:16-alpine').start();
  return { url: c.getConnectionUri(), stop: () => c.stop() };
}

SSE mock

// apps/frontend/src/hooks/__tests__/sse-mock.ts
import { setupServer } from 'msw/node';
import { http, HttpResponse } from 'msw';

export const sseServer = setupServer(
  http.get('/api/stream/agent-run/:id', () => new HttpResponse(
    new ReadableStream({ start(c) { /* emit events */ } }),
    { headers: { 'Content-Type': 'text/event-stream' } },
  ))
);

5 · Risks & Mitigations

  • Testcontainers cold-start > 30s. Mitigation: per-worker shared container; reuse across files via Vitest globalSetup.
  • SSE mock semantics differ from production. Mitigation: contract test against real server nightly.
  • Flaky hook tests due to async timing. Mitigation: use findBy* + fake timers.
