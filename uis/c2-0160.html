1 · Objective

Existing dashboard components and shared UI primitives (loading states, skeletons, animations) are scattered across agentc2/apps/admin/src/components/, agentc2/apps/agent/src/components/, and agentc2/packages/ui/. This epic is the cross-cutting polish pass for the test-status dashboards Filip is building (C2-0173, C2-0179, C2-0189): consolidate primitives into packages/ui, replace inline ad-hoc skeletons with shared <Skeleton/>, animate state transitions consistently, and document the design tokens.

2 · Acceptance Criteria

  • Inventory pass: every Loading* / Skeleton* / inline spinner across the 3 apps catalogued in packages/ui/PRIMITIVES.md.
  • Consolidated <Skeleton/>, <LoadingState/>, <EmptyState/>, <ErrorState/>, <FadeTransition/> exported from @agentc2/ui.
  • Test-status dashboards (C2-0173, C2-0179, C2-0189) migrated to the shared primitives; removed code line-count tracked in PR.
  • Storybook stories per primitive with all visual states; Chromatic baseline captured.
  • Animation tokens (duration, easing) defined in packages/ui/tokens.css; primitives consume them.
  • prefers-reduced-motion respected by all animated primitives.
  • a11y: every primitive passes axe-core; loading states announce via aria-live.
  • Migration guide for other dashboards in packages/ui/MIGRATION.md.

3 · Architecture & Approach

Primitive set

ComponentReplaces
<Skeleton/>ad-hoc .animate-pulse divs in admin + agent apps
<LoadingState/>3 different spinner implementations
<EmptyState/>5+ inline "No data yet" components
<ErrorState/>varied error boxes
<FadeTransition/>inline Framer Motion variants

Token schema

/* packages/ui/tokens.css */
:root {
  --ui-duration-fast: 120ms;
  --ui-duration-base: 240ms;
  --ui-duration-slow: 400ms;
  --ui-easing: cubic-bezier(0.4, 0, 0.2, 1);
}
@media (prefers-reduced-motion: reduce) {
  :root { --ui-duration-fast: 0; --ui-duration-base: 0; --ui-duration-slow: 0; }
}

5 · Risks & Mitigations

  • Migration touches many files. Mitigation: codemod + grep-based replacements; per-app PR.
  • Primitive API too rigid. Mitigation: slot-based + className passthrough.
  • Design-token bikeshedding. Mitigation: Corey sign-off in one 30-min review.
