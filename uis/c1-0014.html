1 · Objective

agentc2/packages/agentc2/src/tools/github-actions-logs.ts ingests GitHub webhook events and apps/agent/src/lib/inngest.ts routes them with retry/dead-letter behavior — the inbound plumbing is live. This epic finalises the 4 specific events the SDLC pipeline reacts to (workflow_run.completed, check_run.completed, pr.opened, pr.merged) into deterministic appello-sdlc state transitions, each with end-to-end Vitest coverage. After this epic, no human watches CI — the pipeline reacts.

2 · Acceptance Criteria

  • 4 Inngest functions, one per event, each idempotent on delivery_id.
  • workflow_run.completed writes a build_runs row tagged with workPackageId derived from PR title prefix.
  • check_run.completed updates the matching build_runs row's per-check status + duration.
  • pr.opened moves the work package to stage review and creates a reviews row owned by sdlc-reviewer.
  • pr.merged moves the work package to stage merge, captures squash SHA, triggers post-merge Playwright via workflow_dispatch.
  • All 4 handlers signature-verified via GitHub's HMAC; replay attacks rejected.
  • Vitest coverage: happy path, missing workPackageId (skip with log), HMAC failure (401), duplicate delivery (no-op), schema-evolution-resilient parser.
  • Dead-letter queue inspected daily; alert if > 5 unhandled in 24h.

3 · Architecture & Approach

Handler structure

// apps/agent/src/lib/inngest.ts (registered functions)
inngest.createFunction(
  { id: 'gh-workflow-run-completed', concurrency: { limit: 10 } },
  { event: 'github/workflow_run.completed' },
  async ({ event, step }) => {
    const wpId = extractWorkPackageId(event.data.pull_requests[0].title);
    if (!wpId) return { skipped: 'no-wp-id' };
    await step.run('upsert-build-run', () => db.buildRun.upsert({ ... }));
  }
);

4 · Implementation Phases (25h)

  1. HMAC verification middleware + delivery-id dedupe (4h).
  2. 4 Inngest functions + DB writes (10h).
  3. Vitest matrix (5 cases per handler = 20 tests) (6h).
  4. Dead-letter queue + alert (3h).
  5. End-to-end smoke against real PR (2h).

5 · Risks & Mitigations

  • Webhook out-of-order delivery. Mitigation: handlers idempotent + state-machine guards (no backward transitions).
  • GitHub event schema evolution. Mitigation: Zod-parsed events; tolerated-unknown-fields; canary test against a recorded event corpus.
  • Lost webhook = stuck work package. Mitigation: reconciler job every 15 min that polls GH for any PR missing recent state transitions.
