1 · Objective

Stand up the cloud + CI/CD + runtime foundation required for the SDLC Flywheel. 3 GPU-enabled AWS compute instances for model inference; VPC, subnets, security groups, IAM service accounts; GitHub Actions → Docker → AWS ECS CI/CD pipeline; observability (logs, metrics, traces).

2 · Acceptance Criteria

  • AWS account, VPC, subnets, security groups, IAM service accounts provisioned via IaC (Terraform or CDK).
  • 3 GPU-enabled instances available for model inference (cost-capped).
  • GitHub Actions → Docker → AWS ECS deploy pipeline green on appello-sdlc main branch.
  • Cost dashboard live (AgentCostDaily) — daily spend visible to the team.
  • Run-and-queue observability live (Inngest job board, structured logs, error pages).

3 · Architecture & Approach

AWS (compute) + Supabase Postgres (state, pgvector) + Inngest (job queue) + Doppler (secrets) + GitHub Actions (CI). The `appello-sdlc` Next.js app is the SDLC control plane; Inngest workers run the agent steps; Supabase holds work_packages_v2 + stage_outputs.
