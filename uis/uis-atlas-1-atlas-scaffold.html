1 · Objective

Stand up the production-grade ATLAS ingestion + retrieval pipeline. Replace the legacy N8n + Google Drive flow with Supabase pgvector storage, OpenAI/Anthropic embeddings, structured chunking (1000-token), and a query API that every downstream agent uses. Foundation for C4 toolboxes.

2 · Acceptance Criteria

  • 729+ meeting transcripts vectorized into 1536-dim embeddings (OpenAI text-embedding-3-small).
  • Ingestion pipeline live in `agentc2/packages/agentc2/src/processors/` driven by Inngest.
  • RagDocument + RagChunk Prisma models + namespace isolation for multi-tenant readiness.
  • Query API (ragQueryTool in agentc2 toolset) returns top-K chunks <500ms p95.
  • N8n legacy flow decommissioned + documented sunset.

3 · Architecture & Approach

Inngest-driven processors in `agentc2/packages/agentc2/src/processors/`. Documents land in IntegrationConnection (Google Drive, N8n export). Chunker splits to 1000-token RagChunks with overlap. Embeddings written to RagChunk.embedding (vector(1536)). Query path: ragQueryTool → Prisma → pgvector cosine search → top-K → reranker (R2 epic).
