1 · Objective

The Vitest pattern already exists in AgentC2: tests/unit/cursor-tools.test.ts, tests/unit/inngest-functions.test.ts, tests/unit/pipeline-supervisor.test.ts. This epic adds the equivalent unit tests for the code-review service (the C1-0026 POST /review handler + its scorers) covering normal/edge/error paths so the autonomous merge gate is provably tested before C1 phase ends.

2 · Acceptance Criteria

  • tests/unit/review-handler.test.ts covers happy-path, idempotent replay, force=true override, partial-tolerance (1 scorer timed out), 4xx auth failure, rate-limit, malformed body.
  • tests/unit/scorers/*.test.ts one file per scorer (tsc, eslint, complexity, unit, playwright, scope); each covers normal + edge (empty diff, binary files, deleted file).
  • Mock contracts: GitHub MCP mocked at the gateway level; scorers stubbed via dependency injection (no real tsc spawn in unit suite).
  • Aggregation logic tested with full weight matrix; rounding behavior covered (0-100 integer, no NaN, no negative).
  • Coverage gate ≥ 90% for review-handler + 85% for scorers.
  • Tests follow the same fixture conventions as the existing 3 suites — no new patterns introduced.
  • All tests run < 30s on dev hardware.

5 · Risks & Mitigations

  • Mocks drift from real GH MCP shape. Mitigation: contract-test that runs nightly against real MCP and updates the mock fixture.
  • Coverage gate game-able. Mitigation: mutation testing (Stryker) on the handler quarterly.
  • Test brittleness on weight changes. Mitigation: aggregation tests parameterized by the playbook constant; one place to update.
