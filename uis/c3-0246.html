1 · Objective

agentc2 RagDocument is the ingestion target; appello/plan/ has draft glossary content; appello-docs/docs/ shows the glossary format. This epic captures 30 construction-domain terms as RagDocument rows so ATLAS retrieval can answer agent questions like "what is a JHA?" / "how is OT calculated in Ontario?" with grounded, definitionally-correct content rather than LLM hallucination.

2 · Acceptance Criteria

  • 30 terms covering: timesheet (5), work orders (5), change orders (4), safety (4), scheduling (4), materials (4), invoicing (4).
  • Each term as { term, definition, synonyms[], domain_examples[], anti_patterns[], source_citations[] }.
  • Definitions written by Nathan, peer-reviewed by Ian.
  • Synonyms field enables retrieval on customer-vernacular (e.g. "toolbox talk" → "safety meeting").
  • Anti-patterns capture common misconceptions (e.g. "OT in Alberta starts at 8 hrs/day OR 44 hrs/week — not both").
  • Source citations: link to authoritative source (provincial labour code, Appello spec, customer interview Fathom).
  • Upserted as RagDocument rows in collection='construction-terms'; one row per term.
  • Retrieval smoke: 10 agent prompts asking "what is X" return the correct term row in top-3.

3 · Architecture & Approach

Term shape

// One row per term
type ConstructionTerm = {
  term: 'Toolbox Talk';
  synonyms: ['safety meeting', 'tailgate meeting', 'JHA briefing'];
  definition: 'A pre-shift safety discussion held by the foreman with the crew covering the day\'s hazards and mitigations. Required by OHSA on most construction sites.';
  domain_examples: ['Discussion of fall protection before roofing work', 'Trench-safety review prior to excavation'];
  anti_patterns: ['Confused with general toolbox inventory check; toolbox talk is a meeting, not an equipment audit.'];
  source_citations: ['https://www.osha.gov/...', 'Owens Insulation interview 2026-03-12 Fathom://...'];
};

4 · Implementation Phases (33h)

  1. Terms inventory (Nathan + Ian) (5h).
  2. Authoring (15h, 30 min/term).
  3. Peer review by Ian (5h).
  4. RagDocument upsert + collection setup (3h).
  5. Retrieval smoke + tuning (3h).
  6. Documentation + handoff to docs.useappello.app integration (2h).

5 · Risks & Mitigations

  • Provincial variance (OT rules vary). Mitigation: per-term notes field documents jurisdictional variants.
  • Outdated as labour code changes. Mitigation: annual review cron + last-reviewed date.
  • Customer-vernacular drift. Mitigation: synonyms updated as new field-interview data lands (C3-0234).
