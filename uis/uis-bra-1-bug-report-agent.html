1 · Objective

Ingest customer-facing bug intake (chat, screenshot, video, Jira ticket), normalize into a structured WorkItem record, attach reproduction steps + suspected files in the Appello monorepo, and hand off to the Planning Agent.

2 · Acceptance Criteria

  • 20-bug validation set: agent correctly extracts ticket title, repro steps, suspected files for ≥85% of cases.
  • Output schema matches `appello-sdlc/prisma/schema.prisma::WorkItem`.
  • Customer chat capture flow live (Slack + intake form).
  • Manual override path: PM can edit any BRA output before Planning Agent picks it up.
  • Cost <$0.10 per processed ticket (model + tool calls).

3 · Architecture & Approach

Mastra agent in `agentc2/packages/agentc2/src/agents/` with tools for Jira read, screenshot OCR (Vision API), and repo file lookup (Cursor MCP). Output written to `WorkItem` in Supabase via the appello-sdlc API. Validation harness in `appello-sdlc/scripts/bra-eval.ts` runs the 20-bug set on every PR.
