1 · Objective

AgentC2's Playwright E2E harness is live: agentc2/playwright.config.ts wires the runner, agentc2/tests-e2e/ holds the spec files (e.g. agent-chat.spec.ts), and agentc2/packages/agentc2/src/mcp/playwright-compression.ts compresses accessibility snapshots so agents can drive the browser without context blow-up. This epic replicates the same install/config against the Appello app surface (today Appello E2E coverage is via screenshot evidence, not yet Playwright-driven), enabling the Spider Testing Swarm to actually execute against the product.

2 · Acceptance Criteria

  • appello/playwright.config.ts created with the same projects/baseURL/timeouts pattern as agentc2.
  • Initial spec set at appello/tests-e2e/: smoke (login, dashboard load, navigation), top-5 flows from C1-0010 baselines.
  • Shared MCP playwright-compression usable from Appello specs (single source, not forked).
  • GitHub Actions workflow appello/.github/workflows/playwright.yml runs on PR; junit artifact named per C1-0012 contract.
  • Test data: per-run seed via Appello's seed-test-tenant.ts script; isolated tenant per worker.
  • Trace recording on failure; uploaded as artifact.
  • Documented in appello/tests-e2e/README.md with local-run instructions.
  • First 10 specs pass green on main; baseline trace captured.

3 · Architecture & Approach

Config alignment with agentc2

// appello/playwright.config.ts
export default defineConfig({
  testDir: './tests-e2e',
  timeout: 30_000,
  use: {
    baseURL: process.env.APPELLO_URL ?? 'http://localhost:3000',
    trace: 'retain-on-failure',
    screenshot: 'only-on-failure',
  },
  reporter: [
    ['list'],
    ['junit', { outputFile: 'playwright-junit.xml' }],
  ],
  projects: [
    { name: 'chromium', use: { ...devices['Desktop Chrome'] } },
    { name: 'mobile-safari', use: { ...devices['iPhone 14'] } },
  ],
});

Shared playwright-compression usage

Specs import compressAccessibilitySnapshot from @agentc2/playwright-compression so Spider-swarm agents can take a snapshot under 4k tokens.

4 · Implementation Phases (30h)

  1. playwright.config.ts + projects + reporter (4h).

  3. Top-5 flows from C1-0010 baselines (10h).

  6. README + local-run + soak (1h).

5 · Risks & Mitigations

  • Appello auth complexity (real OIDC). Mitigation: storageState fixture from a programmatic-login flow.
  • Mobile-Safari project flaky. Mitigation: keep behind a flag for first month; opt-in.
  • Test-tenant seed slow. Mitigation: snapshot-based seeding once stable.
