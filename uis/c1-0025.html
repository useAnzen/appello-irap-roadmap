1 · Objective

AgentC2 cost dashboards are live in the admin app (CostEvent + AgentCostDaily feeds + AgentRun signal). This epic adds the appello-sdlc-side overlay: a per-WP cost timeline (built from transition_logs joined to CostEvent) and alert rules for failed builds, blown budgets, and stuck gates — surfaced in the same shell as the rest of the SDLC dashboard so the operator never has to leave it.

2 · Acceptance Criteria

  • Cost timeline component shows per-stage spend across the work-package lifetime; stacked area by agent.
  • Aggregate KPI: $/merged-hamburger last 30d trend (re-uses C1-0008 metric definition).
  • Alert rules: build failure rate >20% per 24h, stage-budget overrun >3× estimate, gate stuck >48h.
  • Alerts delivered to Slack #sdlc-alerts via existing Inngest webhook; deduped per alertKey on a 24h window.
  • Mute controls per alert rule (per-WP override + global mute).
  • Alert history page at /quality/alerts with ack / resolve workflow.
  • All alert state stored in new model Alert { id, ruleId, severity, payload, ackedBy?, ackedAt?, resolvedAt? }.

3 · Architecture & Approach

Cost timeline data join

transition_logs (workPackageId, stage, enteredAt, exitedAt)
   ⨝ CostEvent (workPackageId, agentId, amountUsd, at)
GROUP BY stage, agent

Alert rule engine

// apps/agent/src/lib/inngest.ts (new function)
inngest.createFunction(
  { id: 'sdlc-alert-eval' },
  { cron: '*/5 * * * *' },
  async ({ step }) => {
    const rules = await db.alertRule.findMany({ where: { enabled: true } });
    for (const rule of rules) {
      const fired = await evalRule(rule);
      if (fired) await step.run('post', () => postToSlack(rule, fired));
    }
  }
);

5 · Risks & Mitigations

  • Alert fatigue. Mitigation: dedupe per alertKey on 24h window; severity-based delivery (only criticals page on-call).
  • Cost-attribution gaps if CostEvent lacks workPackageId. Mitigation: CI lint from C1-0008 prevents new gaps; backfill best-effort by agent+timestamp.
  • Slack outage swallows alerts. Mitigation: Alert rows persist; in-app banner shown until acked even if Slack delivery failed.
