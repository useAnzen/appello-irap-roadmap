1 · Objective

agentc2/packages/agentc2/src/tools/supabase-wp-tools.ts manages Supabase work-package state; the APPELLO-XRM/apps/frontend/src/lib/supabase/ wrappers are the pattern for per-branch schema isolation. Branch-env tooling is partial today. This epic provisions 3 isolated Supabase schemas (or a per-PR schema with a 3-concurrent cap) so parallel factory builds don't collide on the shared dev DB, which is the hard blocker for parallelising the autonomous hamburger factory.

2 · Acceptance Criteria

  • Schema provisioner: provisionSchema({ prNumber, branch }) → { schemaName, connectionUrl, cleanup() }.
  • Schema name pattern: pr_${prNumber}_${shortSha}; truncated to Postgres 63-char identifier limit.
  • Provisioner clones public schema structure + runs prisma migrate deploy against the new schema.
  • Concurrent provisioning cap = 3; over-cap requests queue with FIFO; ETA returned.
  • Seed data optionally applied via seedSchema(schemaName, fixtureSet) for predictable test data.
  • Cleanup on PR close drops the schema cleanly; nightly cron drops any schema > 14d old as backstop.
  • All RLS policies replicated to the per-branch schema (no security degradation in preview).
  • Cost monitor: schema count + storage gauge in C1-0025 alerts.

3 · Architecture & Approach

Provisioner

// agentc2/packages/agentc2/src/tools/supabase-schema-provisioner.ts (new)
import { createClient } from '@supabase/supabase-js';

export async function provisionSchema({ prNumber, branch, shortSha }: Args) {
  await acquireConcurrencyToken();          // caps at 3
  const schemaName = `pr_${prNumber}_${shortSha}`.slice(0, 63);
  const admin = createClient(SUPABASE_URL, SERVICE_ROLE);
  await admin.rpc('create_schema_clone', { src: 'public', dst: schemaName });
  await runPrismaMigrateDeploy({ schemaOverride: schemaName });
  await replicateRlsPolicies(schemaName);
  return {
    schemaName,
    connectionUrl: `${POSTGRES_URL}?schema=${schemaName}`,
    cleanup: () => dropSchema(schemaName),
  };
}

create_schema_clone Postgres function

CREATE OR REPLACE FUNCTION create_schema_clone(src text, dst text) RETURNS void AS $$
DECLARE r record; BEGIN
  EXECUTE format('CREATE SCHEMA IF NOT EXISTS %I', dst);
  FOR r IN SELECT tablename FROM pg_tables WHERE schemaname = src LOOP
    EXECUTE format('CREATE TABLE %I.%I (LIKE %I.%I INCLUDING ALL)', dst, r.tablename, src, r.tablename);
  END LOOP;
END $$ LANGUAGE plpgsql SECURITY DEFINER;

  2. Provisioner Node module + concurrency token (10h).
  3. Prisma migrate-deploy with schema override (6h).
  4. Seed fixtures + per-PR seeding (5h).
  5. Cleanup workflow + nightly cron backstop (5h).
  6. Cost monitoring hook (3h).
  7. Integration with C1-0047 deploy-on-PR (6h).

5 · Risks & Mitigations

  • RLS policy drift between public and per-branch schemas. Mitigation: policy export/import script; CI assertion that count matches.
  • Schema-clone slow on large public schema. Mitigation: structure-only clone (not data); ~3s for 50 tables.
  • Concurrency token leak. Mitigation: token TTL 30 min; auto-release on PR-close or timeout.
