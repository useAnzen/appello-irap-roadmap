1 · Objective

appello-sdlc/apps/* already renders work_packages_v2 + stage outputs (screenshots 01–10 in the repo root). The decomposer emits predecessor/successor structure but the UI for browsing that structure doesn't exist yet. This epic builds the reusable component library: tree view, subtask cards, status badges, progress bars, action buttons — designed for the browse experience that C1-0035 (review) and the public-facing decomposition view both reuse.

2 · Acceptance Criteria

  • Component library at apps/frontend/src/components/decomposition/ with: DecompositionTree, HamburgerCard, DependencyBadge, ProgressBar, ActionMenu.
  • DecompositionTree renders parent UIS + children + edges; supports collapse-all/expand-all; up to 100 hamburgers performant.
  • HamburgerCard shows: slug, title, sequence, complexity, status (pending/in-progress/merged/failed), trust score, quick-link to PR.
  • DependencyBadge tooltip lists predecessors + successors with status colors.
  • ProgressBar visualises the 7-stage pipeline progress (re-uses C1-0005 stage taxonomy).
  • ActionMenu per hamburger: View detail, View PR, View UIS, View decomposer output JSON.
  • All components Storybook-documented; visual regression tests via Chromatic.
  • All components a11y-clean (axe-core CI gate).

3 · Architecture & Approach

Component API

type Hamburger = {
  slug: string; title: string; sequenceNumber: number;
  complexity: 'low'|'medium'|'high';
  status: 'pending'|'in_progress'|'merged'|'failed';
  trustScore?: number;
  prUrl?: string; uisUrl?: string;
  predecessors: string[]; successors: string[];
  stage: Stage;
};

type DecompositionTreeProps = {
  parentUisId: string;
  hamburgers: Hamburger[];
  onHamburgerClick?: (slug: string) => void;
  density?: 'comfortable' | 'compact';
};

Performance budget

  • Initial render < 100ms for 30 hamburgers; < 400ms for 100.
  • Virtualised list below 50 hamburgers.
  • Edge rendering throttled via requestIdleCallback.

4 · Implementation Phases (70h)

  1. Component API + types (5h).
  2. DecompositionTree (15h).
  3. HamburgerCard (10h).
  4. DependencyBadge + ProgressBar (8h).
  5. ActionMenu + integration tests (8h).
  6. Storybook stories (8h).
  7. Visual regression + a11y gates (6h).
  8. Performance pass + virtualisation (10h).

5 · Risks & Mitigations

  • Large tree blows perf budget. Mitigation: virtualisation; lazy edge render.
  • Component API churn during C1-0035 integration. Mitigation: API frozen after Storybook merge; changes go through deprecation.
  • Visual regression flakes. Mitigation: deterministic data fixtures per story.
