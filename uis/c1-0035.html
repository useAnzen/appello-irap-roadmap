1 · Objective

Gate 3 (Decomposition Review) in agentc2/docs/sdlc/07-human-gates.md resumes the workflow via workflow_resume; appello-sdlc/apps/frontend/ hosts gate UIs. This epic adds the visual tree on top of Gate 3: hierarchy view of the proposed decomposition (parent UIS at top, hamburgers as children, dependencies as edges), with inline approve/reject/edit + per-hamburger acceptance-criteria editing — bridging C1-0022's flat review into a true hierarchical reviewer.

2 · Acceptance Criteria

  • Tree view at /library/uis/[uisId]/decomposition/tree renders the parent UIS at root and hamburger children below.
  • Each node expandable: shows description, ACs, change surface, dependencies; click-to-edit opens drawer.
  • Edges show dependency direction; cycles highlighted red; orphan hamburgers (no parent UIS link) flagged.
  • Inline approve/reject per hamburger; bulk approve at parent level.
  • Approve/reject decisions composed into Gate 3 payload: { approved: true | rejected: true, perHamburger: { slug: 'approved'|'rejected'|'edited', edits?: {...} } }.
  • Edit drawer uses the same field set as the hamburger schema (slug locked once approved; title/desc/ACs/change-surface editable).
  • Edits persist as a new DecompositionVersion (per C1-0027); audit trail of who-edited-what.
  • Keyboard nav: arrow keys move focus; a approve, r reject, e edit, ? help.

3 · Architecture & Approach

Component tree

<DecompositionTree>
  <UisNode uisId=...>
    <HamburgerNode slug='wo-1' status='approved'>
      <DependencyEdge to='wo-2' />
      <HamburgerDrawer /> (lazy)
    </HamburgerNode>
    ...
  </UisNode>
</DecompositionTree>

Gate 3 payload assembly

// On dispatch click
const payload = {
  approved: !hasAnyRejection(),
  rejected: hasAnyRejection() && allRejected(),
  reorder: hasAnyEdit(),
  perHamburger: hamburgers.map(h => ({
    slug: h.slug,
    status: h.userDecision,
    edits: h.edits,
  })),
  feedback: feedbackText,
};
await mcp.workflow_resume({ runId, stepId: 'decomposition-review-gate', data: payload });

4 · Implementation Phases (55h)

  1. Tree component scaffolding (react-flow or custom) (10h).
  2. Hamburger node + drawer (10h).
  3. Dependency edge rendering + cycle highlight (8h).
  4. Inline approve/reject/edit + bulk operations (10h).
  5. Gate 3 payload assembly + workflow_resume integration (8h).
  6. Edits → new DecompositionVersion writes (6h).
  7. Keyboard nav + a11y (3h).

5 · Risks & Mitigations

  • Tree overwhelms reviewer on large decompositions. Mitigation: collapse-by-default; chevron toggles; pagination for >30 hamburgers.
  • Mid-review decomposer rerun invalidates state. Mitigation: tree pinned to DecompositionVersion; reviewer notified if newer version exists.
  • Workflow_resume timeout on dispatch. Mitigation: optimistic UI + reconciler from transition_logs.
