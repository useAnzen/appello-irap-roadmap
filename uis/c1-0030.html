1 · Objective

verify-tools and test-runner-tools produce the score inputs; the Review model persists the final rubric + score. This epic finalises the rubric itself: weights per signal (ESLint, tsc, Playwright, change-surface fit), the 0-100 mapping, and the merge thresholds (70% first-pass-compile, 90% after-one-iteration). Once published, the rubric is the contract every autonomous merge is judged against.

2 · Acceptance Criteria

  • Rubric document at agentc2/docs/sdlc/rubric.md with: scoring formula, per-signal weights, thresholds, examples.
  • Weights checked into sdlc-reviewer.playbook.json and versioned; any weight change requires playbook major-version bump + sign-off.
  • Merge thresholds: first-pass autonomous merge requires trustScore ≥ 70; after one healing iteration ≥ 90; else human gate.
  • Rubric calibrated against ≥ 50 historical PRs; calibration sheet attached as appendix.
  • Edge cases documented: how to score type-only changes, docs-only PRs, dependency bumps, generated-file changes.
  • Rubric versioned per release; Review.rubricVersion captured so historical comparability survives weight changes.
  • Reviewer agent prompt updated to reference rubric.md as the explicit scoring contract.

3 · Architecture & Approach

Scoring formula

trustScore = round(
  w_tsc        * tscScore +        // weight 0.20
  w_lint       * lintScore +       // weight 0.10
  w_complexity * complexityScore + // weight 0.10
  w_unit       * unitScore +       // weight 0.20
  w_playwright * playwrightScore + // weight 0.25
  w_scope      * scopeScore        // weight 0.15
)
// weights sum to 1.0; each component is 0-100

Thresholds

StatetrustScoreAction
First-pass≥ 70Autonomous squash-merge
First-pass40–69Trigger 1 healing iteration
After heal≥ 90Autonomous squash-merge
After heal< 90Human gate
First-pass< 40Reject + open auditor ticket

Edge cases (sample)

  • Docs-only PR: playwrightScore = N/A; weight redistributed across remaining signals.
  • Generated files (Prisma client, gql codegen): excluded from scopeScore.
  • Dependency bumps: complexityScore capped at 100; tscScore hard-fail = reject.

4 · Implementation Phases (30h)

  1. Historical PR calibration corpus (10h).
  2. Rubric authoring + edge-case examples (8h).
  3. Playbook JSON + version pin (4h).
  4. Reviewer prompt update + smoke (4h).
  5. Sign-off review with Corey (4h).

5 · Risks & Mitigations

  • Weights overfit to historical corpus. Mitigation: 80/20 train/holdout split; report holdout AUC.
  • Threshold gaming by agents. Mitigation: mutation testing on rubric; surprise-PR audit weekly.
  • Edge cases incomplete. Mitigation: rubric.md kept as living doc; new edge cases triaged within 1 week.
