1 · Objective

agentc2/apps/admin/ has agent dashboards; agentc2/apps/agent/src/components/ hosts shared components; AgentRun + AgentRunTurn carry the activity data. This epic builds the live agent-status panel that reads those models in near-real-time so platform operators see what every agent is doing across the entire fleet at a glance — failure modes, throughput, queue depth — and can intervene before downstream pipelines stall.

2 · Acceptance Criteria

  • Panel at /admin/agents/status: one row per registered agent.
  • Per-row columns: slug, model, in-flight runs count, queue depth, p95 latency (last 5 min), success rate (last 1h), last error timestamp.
  • Click → drill-down to last 50 AgentRun rows + their AgentRunTurn traces.
  • Updates live via the C2-0160 panel primitives + the C2-0158 queue metrics feed.
  • Filter chips: model, status, owner.
  • Quick-action: Pause agent (sets registry flag; queue continues; new tasks rejected).
  • Quick-action: Replay failed run (re-queues with same input).
  • Page Lighthouse perf ≥ 90 with 30 agents in-flight.

3 · Architecture & Approach

Data fetching

// apps/admin/src/app/agents/status/page.tsx
const agents = await db.agent.findMany({
  where: { archived: false },
  include: {
    _count: { select: { invocations: { where: { status: 'pending' } } } },
    metrics: { where: { day: today() }, take: 1 },
  }
});
// p95 latency from AgentMetrics; rest from invocation counts

Pause + replay

POST /api/agents/:slug/pause   // sets Agent.paused = true
POST /api/agents/:slug/replay/:runId   // re-queues with same input + new attempt counter

4 · Implementation Phases (35h)

  1. Page + data fetch (6h).
  2. Drill-down with AgentRunTurn trace (8h).
  3. Live update integration (4h).
  4. Pause + Replay actions (8h).
  5. Filter chips + perf pass (5h).
  6. End-to-end smoke (4h).

5 · Risks & Mitigations

  • Pause causes cascading failures downstream. Mitigation: confirm-modal lists downstream pipelines affected.
  • Replay floods queue. Mitigation: per-replay rate limit + bulk-replay batch with backoff.
  • Trace data large. Mitigation: lazy-load full trace; first paint shows headers only.
