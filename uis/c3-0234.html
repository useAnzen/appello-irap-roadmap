1 · Objective

appello-sdlc/bug-report-agent collects customer issues; Fathom MCP captures interviews; agentc2 RagDocument/RagChunk indexes them. This epic is Ian's customer-interview series: Ian interviews 12 customers, categorizes the bug + feature signals, and persists structured rows into RagDocument so ATLAS retrieval surfaces real customer evidence to the decomposer + auditor at runtime.

2 · Acceptance Criteria

  • ≥ 12 customer interviews completed across the next 6 weeks; transcripts in Fathom tagged customer:<slug>.
  • Each interview produces ≥ 5 categorized signals: { customer, signalType: 'bug'|'feature'|'workaround'|'risk', area, body, transcriptUrl, severity }.
  • Signals upserted as RagDocument rows in collection='customer-signals'; chunked at 600 tokens with 80-token overlap.
  • Categorisation done by Ian (manual) + LLM tie-break on ambiguous; precision target 0.9 measured on a 30-row spot check.
  • Bug-report-agent's existing ticket exports also imported into the same collection so this isn't isolated.
  • Decomposer + auditor agent prompts updated to retrieve from customer-signals when scope mentions a customer-impacted area.
  • Quarterly review: low-impact signals archived to keep collection lean.

3 · Architecture & Approach

Interview pipeline

Customer call (Fathom)
  → fathom-meeting-summary workflow runs
  → Ian categorises signals (UI form)
  → upsert RagDocument(collection='customer-signals', body, metadata: { customer, area, severity })
  → embedding via existing rag-tools.upsertDocument
  → available to decomposer/auditor via retrievePrecedent({ collection: 'customer-signals' })

4 · Implementation Phases (27h)

  1. Interview booking + cadence (3h).
  2. Signal categorisation UI (5h).
  3. RagDocument ingestion path (3h).
  4. 12 interviews + categorisation (10h).
  5. Bug-report-agent import sub-flow (3h).
  6. Decomposer/auditor prompt update + smoke (3h).

5 · Risks & Mitigations

  • Customer privacy. Mitigation: scrub PII at ingestion; only generalised signals persisted.
  • Signal noise dominates. Mitigation: severity gate; only high+med ingested.
  • Interview scheduling slip. Mitigation: async written-survey backup option for 4 of the 12.
