1 · Objective

agentc2/packages/agentc2/src/tools/cursor-tools.ts dispatches Cursor Cloud builds; merge-deploy-tools.ts handles the merge/deploy lifecycle; sandbox-tools.ts (deprecated v4) used to manage ephemeral envs. Per agentc2/docs/sdlc/03-hamburger-factory.md the v4 factory moved to GitHub CI on the PR branch instead of sandboxes. This epic completes the ephemeral-staging-deploy-on-PR pattern in the new architecture: every PR auto-deploys to a per-PR Vercel preview + provisions a per-branch Supabase schema (C1-0048) so the sdlc-reviewer's Playwright runs hit a real environment, not a mock.

2 · Acceptance Criteria

  • GitHub Actions workflow ci-staging-deploy.yml runs on PR open/sync; deploys frontend to Vercel preview URL.
  • Per-branch Supabase schema provisioned via the C1-0048 tooling; SUPABASE_SCHEMA env injected into the Vercel preview.
  • Preview URL posted as PR comment by the workflow; updates on every push.
  • Playwright runs in ci-playwright.yml (C1-0012) target the preview URL via env var.
  • On PR close (merged or not), cleanup job tears down Vercel preview + drops the Supabase schema.
  • Failure to deploy = trust-score penalty (-20) + reviewer comment with logs.
  • sandbox-tools.ts formally deprecated; export removed in next major release.
  • Documentation: agentc2/docs/sdlc/v4-deploy-on-pr.md covers the new pattern + migration guide from v3 sandboxes.

3 · Architecture & Approach

v4 deploy-on-PR flow

PR opened → ci-staging-deploy.yml
  ├─ provision per-branch Supabase schema (C1-0048)
  ├─ deploy Vercel preview with SUPABASE_SCHEMA env
  ├─ comment preview URL on PR
  └─ trigger ci-playwright.yml against preview URL
PR closed → ci-cleanup.yml
  ├─ delete Vercel preview
  └─ drop Supabase schema

Migration from v3 sandboxes

v3 (sandboxes)v4 (deploy-on-PR)
sandbox-tools.ts provisions envGH Actions provisions env
Agent SSHs into sandboxCursor Cloud pushes to PR branch
Reviewer hits sandbox URLReviewer hits Vercel preview URL
Manual cleanupAuto-cleanup on PR close

5 · Risks & Mitigations

  • Vercel preview quota exhausted. Mitigation: per-org quota monitoring; auto-cleanup of preview > 14d old.
  • Per-branch Supabase schemas leak. Mitigation: cleanup nightly cron as backstop to PR-close cleanup.
  • Preview URL not ready when Playwright runs. Mitigation: workflow waits for Vercel ready ping before Playwright job.
