1 · Objective

AgentTrigger / TriggerEvent / AgentAlert in the AgentC2 schema are the existing primitives for event-driven notifications; Review is the queue. This epic extends the trigger system to fire (1) in-app badges, (2) Slack #sdlc-reviews messages, and (3) end-of-day digest emails when a Review row enters PENDING with an AI-authored PR. Goal: zero unreviewed AI PRs lingering > 4 hours.

2 · Acceptance Criteria

  • AgentTrigger rule: when Review.status transitions to PENDING AND Review.author = 'agent:cursor-coding-agent', fire TriggerEvent.
  • Three notification channels register listeners: in-app badge (live counter in sidebar), Slack DM to assigned reviewer + #sdlc-reviews, daily digest email at 8am local.
  • In-app badge shows count of pending AI reviews on the Reviews nav item; click → filtered list.
  • Slack messages include PR link, trust score, expected merge gate; react :white_check_mark: on Slack triggers approve.
  • Digest email groups by reviewer; sent only if reviewer has ≥1 pending; opt-out per user.
  • Notification dedupe: same Review fires at most 1 Slack + 1 badge increment, but stays in digest until resolved.
  • AgentAlert row created when a pending review is unreviewed > 4h; escalates to manager Slack DM.

3 · Architecture & Approach

Trigger rule

// agentc2/seeds/sdlc-triggers.ts
await db.agentTrigger.upsert({
  where: { slug: 'review-pending-ai' },
  create: {
    slug: 'review-pending-ai',
    sourceModel: 'Review',
    onChange: { field: 'status', from: '*', to: 'PENDING' },
    where:    { authorMatches: 'agent:cursor-coding-agent' },
    channels: ['in-app-badge', 'slack:#sdlc-reviews', 'slack:dm-assignee', 'digest-email'],
    escalation: { afterMinutes: 240, channel: 'slack:dm-manager' },
  }
});

Channel implementations

  • in-app-badge — SSE event from TriggerEvent → React Query invalidation → badge updates.
  • slack — Slack MCP chat.postMessage + reaction-listener webhook.
  • digest-email — daily cron groups by reviewer; uses Postmark.

4 · Implementation Phases (28h)

  1. AgentTrigger rule seeding (3h).
  2. In-app badge + SSE wiring (5h).
  3. Slack channel impl + reaction-listener for approve (8h).
  4. Daily digest email cron (4h).
  5. Escalation rule (4h).
  6. Opt-out preferences + dedupe (4h).

5 · Risks & Mitigations

  • Slack rate limits. Mitigation: batch under burst load; per-channel rate-limit budget.
  • Notification fatigue. Mitigation: dedupe per Review; digest collapses multiple pendings; opt-out granular.
  • Reaction-as-approve security. Mitigation: reactor must be in the appello-reviewers Slack group; second-factor via in-app confirm.
