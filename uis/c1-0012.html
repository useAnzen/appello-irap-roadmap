1 · Objective

agentc2/packages/agentc2/src/tools/verify-tools.ts is the AgentC2 toolbox that sdlc-pipeline-supervisor.ts uses to call wait-for-checks on GitHub before allowing autonomous merge. The polling contract is live in AgentC2. This epic completes the matching half on the appello-sdlc side: the GitHub Actions workflow definitions that produce the named checks the supervisor waits for (tsc, eslint, vitest, playwright-affected, security-scan), with the right names, the right artifacts, and the right exit-code semantics.

2 · Acceptance Criteria

  • 5 named GitHub Actions checks on every PR to appello-sdlc: build/tsc, lint/eslint, test/vitest, test/playwright-affected, security/secret-scan.
  • Check names exactly match the strings hard-coded in verify-tools.ts wait-for-checks — no fuzzy matching.
  • Each workflow uploads a structured artifact (junit, sarif) named per the supervisor's expected pattern.
  • playwright-affected uses pull_request file-diff to run only specs touched (or specs covering touched stories per C1-0004); reports skipped count.
  • All workflows runnable locally via act; documented in appello-sdlc/CONTRIBUTING.md.
  • Branch protection enforces all 5 checks must pass before merge — supervisor's auto-merge never bypasses.
  • End-to-end smoke: AgentC2 launches a sample hamburger; supervisor's wait-for-checks resolves to success after the real workflows complete.

4 · Implementation Phases (45h)

  1. Workflow file authoring (12h) — 5 workflows with matching names + artifacts.
  2. Affected-spec selector script (8h).
  3. Branch protection + admin sign-off (4h).
  4. act local-runnability + CONTRIBUTING docs (4h).
  5. End-to-end smoke with real AgentC2 hamburger dispatch (10h).
  6. Failure-mode drills (timeouts, partial completes, GH outage) (7h).

5 · Risks & Mitigations

  • Check-name drift between AgentC2 and Actions YAML. Mitigation: single shared constants file generated by codegen into both repos.
  • Playwright affected-spec false negatives. Mitigation: nightly full-suite cron; failures escalate.
  • GH Actions queue starvation. Mitigation: larger runner pool for SDLC; queue-depth alert.
