1 · Objective

agentc2/packages/agentc2/src/tools/workflow-crud-tools.ts + agent-crud-tools.ts are the AgentC2-side CRUD tools; appello-sdlc/apps/frontend/src/lib/actions/router/actions.ts is the appello-sdlc CRUD surface. This epic writes the Vitest coverage for actions.ts using the isolation conventions documented in the appello-xrm-test-isolation skill (testcontainers Postgres, withTx() rollback, assert-safe-db guards, no destructive patterns). After this epic every CRUD route has paired tests against a real-DB.

2 · Acceptance Criteria

  • Test suite at apps/frontend/src/lib/actions/router/__tests__/actions.test.ts; one describe per CRUD route.
  • Each test runs against a testcontainers Postgres started by the shared db:test:up harness; isolated per test via withTx() rollback.
  • Tests use the assert-safe-db guard so a stray DROP TABLE in a fixture aborts CI rather than nuking dev data.
  • Coverage: happy path, validation failure (matches C1-0002 Zod contract), missing FK, optimistic-concurrency conflict, idempotent replay.
  • No prisma.$queryRaw outside an explicit destructive_test_patterns: false annotation.
  • Vitest watch mode runs all tests < 60s on dev hardware.
  • Coverage gate ≥ 85% lines for actions.ts in CI.

3 · Architecture & Approach

Isolation pattern (from appello-xrm-test-isolation)

// apps/frontend/src/lib/actions/router/__tests__/actions.test.ts
import { withTx, getTestDb } from '@appello/test-isolation';
import { actions } from '../actions';

describe('actions.createWorkPackage', () => {
  it('creates a WP with valid payload', () =>
    withTx(async (tx) => {
      const wp = await actions.createWorkPackage({ slug: 'wp-1', title: 'T' }, { db: tx });
      expect(wp.id).toBeDefined();
    }));
  it('rejects invalid slug with 400', () => ...);
});

Safety guards

  • db:test:up brings up a testcontainers Postgres bound to a non-prod port.
  • .env.test isolated; never reads from .env.
  • assert-safe-db Vitest setup hook fails the suite if DATABASE_URL contains a prod-shaped host.

4 · Implementation Phases (20h)

  1. db:test:up harness verified locally + CI (3h).
  2. Test scaffolding (testcontainers, withTx) (4h).
  3. Per-route tests across 12 CRUD routes (10h).
  4. Coverage gate + CI integration (3h).

5 · Risks & Mitigations

  • testcontainers slow in CI. Mitigation: shared container per worker; warm-pool.
  • Flaky FK tests if seed data changes. Mitigation: fixtures live in __fixtures__/ and are loaded per test.
  • Skill conventions drift. Mitigation: conventions imported from @appello/test-isolation; one place to update.
