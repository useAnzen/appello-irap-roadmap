1 · Objective

agentc2/docs/sdlc/00-overview.md is the existing architecture reference; appello-roadmap/agent-planning-guidelines.md is the cross-component planning surface. This epic establishes recurring architecture reviews (Corey-led) using these docs as the working surface — bi-weekly 90 min sessions that keep the system shape coherent as the autonomous factory adds primitives and the team grows.

2 · Acceptance Criteria

  • Cadence: bi-weekly Thursday 90 min; recurring meeting on shared calendar.
  • Agenda template at appello-roadmap/architecture-reviews/AGENDA-TEMPLATE.md: 30 min what-changed-since-last, 30 min discussion of pending architecture proposals, 30 min decisions + ADR drafting.
  • Each review produces ≥ 1 ADR (Architecture Decision Record) committed to appello-roadmap/adr/NNNN-title.md or explicit "no decisions" note.
  • ADR format: Context / Decision / Status (proposed/accepted/superseded) / Consequences / Alternatives considered.
  • 00-overview.md + agent-planning-guidelines.md updated within 1 week of any accepted ADR.
  • First 4 review meeting notes published as proof-of-cadence.
  • Cross-team visibility: ADRs cross-posted to #architecture Slack channel.

4 · Implementation Phases (35h)

  1. Cadence + calendar setup (3h).
  2. ADR template + tooling (4h).
  3. Architecture-review directory + initial population (4h).
  4. First 4 sessions facilitated (16h).
  5. Doc-update enforcement workflow (4h).
  6. Cross-team visibility (Slack post, ADR index page) (4h).

5 · Risks & Mitigations

  • Meetings become status-recital. Mitigation: what-changed segment time-capped 30 min; rest is decision-driven.
  • ADRs not actually consulted. Mitigation: ADR index linked from CLAUDE.md; agent prompts reference accepted ADRs.
  • Architecture drift between reviews. Mitigation: linting rule: PR touching >5 files in agentc2/packages/agentc2/src/agents/ requires ADR link.
