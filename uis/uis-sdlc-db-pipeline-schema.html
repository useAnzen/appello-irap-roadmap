1 · Objective

Build the `appello-sdlc` Postgres/Prisma schema and API surface that tracks every work package as it moves through the 7-stage pipeline. Define tables, FK integrity, idempotent migrations, and a Next.js App-Router API for CRUD + agent state transitions.

2 · Acceptance Criteria

  • 8+ Prisma models live: WorkPackageV2, WorkItem, StageOutput, Review, BuildRun, TestRun, DocArtifact, TransitionLog (+ FKs/indexes).
  • Idempotent migration framework with safe rollback (see appello-xrm-prisma-migrations skill).
  • Next.js API routes for CRUD on every model + agent status endpoints.
  • Zod validation on every API route; unit tests for FK + referential integrity.
  • appello-sdlc Next.js dashboard live: stage-board, work-package detail, transition log.

3 · Architecture & Approach

Next.js 16 App Router app in `appello-sdlc/`. Prisma schema in `appello-sdlc/prisma/schema.prisma`. API routes in `appello-sdlc/app/api/`. Frontend in `appello-sdlc/app/(dashboard)/`. All other SDLC agents talk to this app via REST. Migration discipline enforced by `assert-safe-db.ts` script in CI.
