1 · Objective

agentc2/Dockerfile + agentc2/DEPLOY.md describe the production stack (DigitalOcean compose + Caddy). The stack is reproducible from docker-compose. This epic re-expresses it as proper Terraform modules so a new environment (staging, customer-isolated) can be stood up in minutes rather than hours of manual DO console clicks, with state in S3-compatible storage so the team can collaborate.

2 · Acceptance Criteria

  • Terraform root module at infra/terraform/agentc2/ with vars: env_name, droplet_size, postgres_size, domain.
  • Submodules: compute (DO droplet + cloud-init), db (managed Postgres), storage (Spaces bucket), network (VPC + firewall), dns (Cloudflare records), tls (Caddy LetsEncrypt config).
  • State backend: DO Spaces (S3-compatible); locked via DynamoDB-equivalent (state-lock-store pattern).
  • terraform apply from scratch provisions a working agentc2 env < 15 min.
  • Doppler secret integration via terraform-doppler-provider; no secrets in tf state.
  • Idempotent: re-apply on unchanged config is a no-op (drift detected and reported).
  • Documentation: infra/terraform/agentc2/README.md with example envs (dev, staging, prod) and bootstrap steps.
  • CI: terraform validate + terraform plan on every PR touching infra/.

3 · Architecture & Approach

Module layout

infra/terraform/agentc2/
  main.tf              # composes submodules
  variables.tf
  outputs.tf
  backend.tf           # DO Spaces state
  modules/
    compute/           # DO droplet + cloud-init
    db/                # managed Postgres
    storage/           # Spaces bucket
    network/           # VPC + firewall
    dns/               # Cloudflare records
    tls/               # Caddy config
  envs/
    dev.tfvars
    staging.tfvars
    prod.tfvars

compute module — droplet user-data

# modules/compute/cloud-init.yml
runcmd:
  - curl -fsSL https://get.docker.com | sh
  - mkdir -p /opt/agentc2 && cd /opt/agentc2
  - curl -fsSL ${docker_compose_url} -o docker-compose.yml
  - doppler run -p agentc2 -c ${env} -- docker compose up -d

5 · Risks & Mitigations

  • DO provider quirks vs AWS. Mitigation: stick to digitalocean_droplet + digitalocean_database_cluster; avoid bleeding-edge resources.
  • Drift between Terraform and docker-compose. Mitigation: compose file URL pinned by Git SHA in Terraform var.
  • State file accidental delete. Mitigation: Spaces versioning enabled; weekly state backup.
