1 · Objective

agentc2/packages/ui/src/components/ai-elements/diff.tsx is the existing diff component; agentc2/scripts/create-test-user.ts + create-test-tenant.ts seed test data; appello-sdlc has the PR-review primitives. This epic extends the diff component with line-level commenting + AI-suggested fixes so the C4 Apex catalog ships a first reviewer-augmentation skill: paste a PR diff, get inline comments + suggested edits, accept or reject — usable both within Appello-internal workflows and as a published Apex skill.

2 · Acceptance Criteria

  • <DiffWithComments/> extends <Diff/>: hover-to-add-comment per line; comments stored in ApexRun.output as structured JSON.
  • AI-suggested comments mode: agent produces inline comments per line; user accepts/rejects each.
  • Suggested-fix component: each comment can carry a code-replacement; one-click apply produces updated diff.
  • Apex skill registration: review-pr-diff registered in ApexSkill with parameter schema { diff: string, language?: string, focusAreas?: string[] }.
  • Skill billed per run via C4-0340 ApexBillingEvent.
  • Embeddable on customer pages via <ApexEmbed skill='review-pr-diff' />.
  • Performance: 500-line diff renders + receives initial AI comments within 8s p95.
  • Storybook coverage + Chromatic baseline.

3 · Architecture & Approach

Component extension

// packages/ui/src/components/ai-elements/diff-with-comments.tsx
export function DiffWithComments({
  diff, suggestedComments, onComment, onAcceptFix
}: DiffWithCommentsProps) {
  return (
    <Diff diff={diff}>
      {(lineProps) => (
        <Line {...lineProps}>
          <CommentColumn
            comments={commentsForLine(lineProps.lineNo)}
            onAdd={onComment}
            onAcceptFix={onAcceptFix}
          />
        </Line>
      )}
    </Diff>
  );
}

Skill flow

POST /api/apex/skills/review-pr-diff/run
  → enqueue ApexRun
  → reviewer agent generates inline comments
  → return as { comments: [{ lineNo, body, suggestedFix? }] }
  → UI renders via DiffWithComments
  → ApexBillingEvent created on success

5 · Risks & Mitigations

  • Diff component prop-drilling for comments. Mitigation: render-prop API (above); composable.
  • Suggested-fix conflicts with subsequent comments. Mitigation: recompute against current diff state; flag stale.
  • Hallucinated suggestions. Mitigation: agent prompt grounded in detected language + lint rules; conf score per comment.
