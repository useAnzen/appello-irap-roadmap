1 · Objective

claude-for-appello/commands/qa-author-flow.md captures the pattern for codifying domain learnings as a reusable test/spec; AgentC2's agent-learning-tools.ts is the channel for feeding corrections back as prompt grounding. This epic builds the structured pipeline from Nathan's domain-correction notes (C1-0039) into the decomposer's runtime context so corrections compound rather than vanish.

2 · Acceptance Criteria

  • Each Nathan finding written as a structured AgentLearning row: { agentSlug, topic, scenarioId, before, after, rationale, createdBy: 'nathan-helm' }.
  • Learnings retrieved at decomposer-prompt assembly time via retrieveLearnings(agentSlug, topic); injected into the system prompt as a <construction-lessons> block.
  • Cap: 20 most-recent + 10 highest-impact learnings injected (to keep tokens bounded); selection scored by recency × outcome-correlation.
  • Each learning has a measurable impact score updated after each downstream decomposer run: +1 if grounded run improves, -1 if regresses, 0 if unchanged.
  • Low-impact learnings (impact ≤ -3) auto-archived (kept in DB, removed from prompt).
  • Workflow: Nathan finds gap → submits via /learnings/new form OR qa-author-flow command → review by Ian → learning persisted.
  • Cross-agent: same channel can be used for sdlc-auditor and sdlc-reviewer learnings (not only decomposer).

3 · Architecture & Approach

Learning shape

model AgentLearning {
  id          String   @id @default(cuid())
  agentSlug   String
  topic       String   // 'construction-gaps' | 'decomposition-pitfalls' | ...
  scenarioId  String?
  before      String   // "what the agent did wrong"
  after       String   // "what it should have done"
  rationale   String   // domain reasoning
  createdBy   String
  createdAt   DateTime @default(now())
  archived    Boolean  @default(false)
  impact      Int      @default(0)
  runsScored  Int      @default(0)
  @@index([agentSlug, archived, impact])
}

Prompt assembly

// agentc2/packages/agentc2/src/agents/uis-decomposer.ts (modified)
const learnings = await agentLearning.retrieve({
  agentSlug: 'uis-decomposer',
  topics: ['construction-gaps', 'decomposition-pitfalls'],
  limit: 30,
  archived: false,
  rankBy: ['recency', 'impact'],
});
const systemPrompt = `${BASE_PROMPT}\n\n<construction-lessons>\n${learnings.map(formatLearning).join('\n')}\n</construction-lessons>`;

  6. Cross-agent generalisation (5h).

5 · Risks & Mitigations

  • Prompt-bloat from too many learnings. Mitigation: hard 30-item cap; token budget audit on every prompt.
  • Conflicting learnings. Mitigation: conflict-detect (semantic overlap + opposite advice) flagged at submission.
  • Impact scoring noisy on small samples. Mitigation: minimum runsScored ≥ 5 before auto-archive.
