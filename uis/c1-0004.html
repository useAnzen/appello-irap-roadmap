1 · Objective

The sdlc-reviewer agent (orchestrated by sdlc-pipeline-supervisor.ts) already produces a trust score on every Cursor-generated PR using tsc + ESLint signals. This epic extends the acceptance criteria with two production-grade gates: (1) the Playwright E2E suite from agentc2/tests-e2e/ must be green for affected user-stories; (2) the actual change surface in the PR diff must be a subset of the changeSurface declared in the hamburger manifest. Both signals feed back into the reviewer's trust score so auto-merge thresholds can be tightened without false negatives.

2 · Acceptance Criteria

  • Reviewer agent ingests Playwright junit XML from the CI artifact tests-e2e-results.xml and maps each suite to user-story IDs via tests-e2e/*.spec.ts tags.
  • Affected stories computed from the hamburger's userStories[]; any failure or skip in the affected slice = trust-score penalty (-30 baseline, configurable).
  • Change-surface gate: git diff --name-only base..head ⊆ changeSurface.new ∪ changeSurface.modified; any out-of-scope file = -40 penalty + reviewer comment.
  • Trust-score weights documented and version-pinned in sdlc-reviewer.playbook.json.
  • Auto-merge threshold (replacement of human Gate 6) reads from autoApprovePrBelow set at Gate 4; default = manual review when either new gate fails.
  • Reviewer posts an itemized PR comment naming the failing gate and the specific files/tests.

3 · Architecture & Approach

Two new inputs to the trust-score function in sdlc-pipeline-supervisor.ts:

// playwrightGate
const affectedSpecs = playwrightResults.filter(s => s.tags.some(t => hamburger.userStories.includes(t)));
const playwrightPenalty = affectedSpecs.some(s => s.status !== 'passed') ? WEIGHTS.playwrightFail : 0;

// scopeGate
const declared = new Set([...hamburger.changeSurface.new, ...hamburger.changeSurface.modified]);
const outOfScope = prDiff.filter(f => !declared.has(f));
const scopePenalty = outOfScope.length ? WEIGHTS.outOfScope : 0;

const trust = base + tscScore + eslintScore - playwrightPenalty - scopePenalty;

This is the gate logic that allows the v4 architecture (see C1-0003) to remove human Gates 5–7 safely.

4 · Implementation Phases (27h)

  1. Playwright tag convention + spec audit (5h) — every spec gets a @story:UIS-XX-S1 tag; junit reporter wired in playwright.config.ts.
  2. Story-to-spec mapper + reviewer ingestion (8h) — parse junit XML, intersect with hamburger.userStories.
  3. Change-surface gate logic (5h) — git diff parsing + set diff against manifest.
  4. Trust-score weight tuning + playbook version pin (4h) — empirical calibration against last 20 PRs.
  5. PR comment template + end-to-end smoke on 3 recent hamburgers (5h).

5 · Risks & Mitigations

  • Playwright flakes treated as real failures. Mitigation: 1 retry built into reporter; flake-rate dashboard separately tracked.
  • Legitimate refactors trigger scope penalty. Mitigation: hamburger author can declare changeSurface.adjacent[] for known-touched files; reviewer treats as soft penalty.
  • Trust-score thresholds drift after model upgrades. Mitigation: per-release recalibration via golden PR set.
