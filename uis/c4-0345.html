1 · Objective

agentc2/packages/agentc2/src/sessions/manager.ts + agentc2/packages/agentc2/src/networks/ already handle agent message handling; agentc2/apps/agent/src/lib/inngest.ts is the queue. This epic implements per-agent-type message handlers on top of those primitives so Apex skills can be invoked from chat / Slack / API with the right routing logic per agent type — code-reviewer routes to the diff toolchain (C4-0344), domain-expert routes to ATLAS retrieval (C3 stack), etc. Without this routing layer, every Apex skill rolls its own dispatch.

2 · Acceptance Criteria

  • Handler registry at agentc2/packages/agentc2/src/handlers/: one file per agent type (code-reviewer, domain-expert, planner, healer, classifier).
  • Each handler: { matches(message): boolean, handle(message, session): Promise<Reply> }.
  • Router invoked from sessions/manager.ts: tries handlers in priority order; first match wins.
  • Common middleware: rate-limit, cost-cap, audit log, retry; applied uniformly.
  • Handler dispatch tracked in AgentInvocation (C1-0021) with handlerSlug.
  • Fallback handler when no match: returns structured "no skill matches" + suggests closest agent.
  • Vitest coverage per handler matches C1-0023 conventions.
  • End-to-end smoke: 5 sample messages routed to 5 different handlers correctly.

3 · Architecture & Approach

Handler shape

// agentc2/packages/agentc2/src/handlers/code-reviewer.ts
import type { AgentHandler } from './types';
export const codeReviewer: AgentHandler = {
  slug: 'code-reviewer',
  priority: 100,
  matches: (m) => m.intent === 'review-diff' || /\b(review|critique)\b.*\bdiff\b/i.test(m.text),
  handle: async (m, session) => {
    const diff = extractDiff(m);
    const run = await apexRun.create({ skillSlug: 'review-pr-diff', input: { diff } });
    const result = await reviewerAgent.run(run);
    return reply(result);
  }
};

Router

const HANDLERS = [codeReviewer, domainExpert, planner, healer, classifier];
export async function route(m: Message, session: Session) {
  for (const h of [...HANDLERS].sort((a,b) => b.priority - a.priority)) {
    if (await h.matches(m)) {
      await audit({ session, handler: h.slug });
      return withMiddleware(h, m, session);
    }
  }
  return fallbackHandler(m, session);
}

5 · Risks & Mitigations

  • Match logic overlapping → wrong handler. Mitigation: priority + audit trail; mismatches in audit trigger handler review.
  • Middleware ordering subtle. Mitigation: ordered list + explicit test of stack order.
  • Fallback gives bad UX. Mitigation: fallback suggests closest agent via embedding similarity, not just literal "no match".
