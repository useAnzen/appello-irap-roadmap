1 · Objective

agentc2/packages/agentc2/src/tools/verify-tools.ts already runs ESLint, tsc, and complexity checks on PRs and the Review model in appello-sdlc persists scores. This epic completes the backend that ties them together: a POST /review handler that intakes a Cursor Cloud PR diff, dispatches all scorers in parallel, aggregates the components into a single 0–100 Review.trustScore, and persists the breakdown so downstream auto-merge logic (C1-0024) has a single source of truth.

2 · Acceptance Criteria

  • POST /api/review handler accepts { prNumber, repository, hamburgerSlug }; returns { reviewId, trustScore, breakdown }.
  • Handler fetches PR diff via GitHub MCP, dispatches verify-tools (tsc, eslint, complexity) + test-runner-tools (unit, playwright) in parallel via Promise.all.
  • Aggregated trust score = weighted sum per sdlc-reviewer.playbook.json; weights versioned with playbook.
  • Breakdown persisted in Review.scoreBreakdown (Json column): { tsc, eslint, complexity, unit, playwright, scope, weights }.
  • Handler idempotent on { prNumber, headSha }; re-runs return the cached review unless force=true.
  • Auth: handler requires a service token; rate-limited at 30 req/min per token.
  • Errors are partial-tolerant: if Playwright scorer times out, score component is null with reason logged, but other components still produce a (partial) review.
  • Vitest coverage matches C1-0023 conventions; end-to-end smoke against the last 5 merged PRs.

3 · Architecture & Approach

Handler outline

// apps/agent/src/app/api/review/route.ts (new)
export async function POST(req: Request) {
  const { prNumber, repository, hamburgerSlug } = await req.json();
  const headSha = await gh.getPrHeadSha(repository, prNumber);
  const existing = await db.review.findUnique({ where: { prNumber_headSha: { prNumber, headSha } } });
  if (existing && !force) return Response.json(existing);

  const diff = await gh.getPrDiff(repository, prNumber);
  const hamburger = await db.workItem.findUnique({ where: { slug: hamburgerSlug } });

  const [tsc, eslint, complexity, unit, playwright] = await Promise.allSettled([
    verifyTools.tsc(diff),
    verifyTools.eslint(diff),
    verifyTools.complexity(diff),
    testRunnerTools.unit(prNumber),
    testRunnerTools.playwrightAffected(prNumber, hamburger),
  ]);
  const scope = verifyTools.scope(diff, hamburger.changeSurface);

  const breakdown = collect(tsc, eslint, complexity, unit, playwright, scope);
  const trustScore = applyWeights(breakdown, playbook.weights);

  return Response.json(await db.review.create({ data: { prNumber, headSha, trustScore, scoreBreakdown: breakdown }}));
}

5 · Risks & Mitigations

  • Long-running scorers block handler. Mitigation: per-scorer timeout (30s default); time-out becomes null component with logged reason.
  • Race when two reviews fire for the same SHA. Mitigation: upsert on unique {prNumber, headSha}; second arrival returns first result.
  • Weight drift between scorers. Mitigation: playbook SHA included in Review.playbookSha.
