1 · Objective

The signals already exist: appello-roadmap/hamburger-manifest-1.34.json defines the release scope; CostEvent + AgentCostDaily in the AgentC2 schema track per-run dollar cost; AgentRun records throughput and outcomes. This epic compiles them into a release-anchored milestone tracker that answers two questions at any moment: (1) what % of release-1.34 hamburgers compiled green on the first build attempt? (2) what is the marginal $ per merged ticket? The tracker is the operating dashboard for the autonomy KPI.

2 · Acceptance Criteria

  • Tracker page at /runs/releases/1.34 reads hamburger-manifest-1.34.json as the scope contract; renders one row per hamburger.
  • Per-row metrics: status (pending/in-progress/merged/failed), first-pass compile (yes/no/n-a), $ cost from CostEvent.sum joined on workPackageId, wall-clock duration from AgentRun.
  • Top-of-page KPIs: First-Pass-Compile % (merged hamburgers only), Mean $/Merged-Ticket, p50 / p95 cycle time, % complete vs plan.
  • Time-series chart: weekly First-Pass-Compile % across last 12 weeks (regression line + target band).
  • $ per ticket excludes infra fixed cost; includes only variable model + API spend (sourced from AgentCostDaily).
  • Tracker auto-refreshes every 60s via SSE; manual refresh button.
  • CSV export for IRAP / NRC audit (one row per hamburger with all metrics).

3 · Architecture & Approach

Data joins

hamburger-manifest-1.34.json (slug, title, sequenceNumber, complexity)
   ⨝ work_packages_v2 ON slug              -- status, dates
   ⨝ AgentRun         ON workPackageId     -- wall clock, attempts
   ⨝ CostEvent        ON workPackageId     -- model + API $
   ⨝ build_runs       ON workPackageId     -- first-pass compile boolean

First-pass-compile definition

For a merged hamburger, first-pass compile = true iff the very first build_runs row for its PR has status='success'. A failed first build (even if eventually green) sets it to false. n-a for hamburgers without a build step (docs-only).

$ per ticket

$/ticket = SUM(CostEvent.amountUsd WHERE workPackageId ∈ release.hamburgers AND state='merged')
          / COUNT(distinct workPackageId WHERE state='merged')

5 · Risks & Mitigations

  • Cost attribution drift if CostEvent.workPackageId is missing on some runs. Mitigation: CI lint that fails any agent run that emits a CostEvent without workPackageId.
  • Manifest scope changes mid-release skew %-complete. Mitigation: snapshot manifest at release-cut date; tracker uses snapshot, not live manifest.
  • Materialized view staleness. Mitigation: 60s refresh job; lag indicator visible on the page.
